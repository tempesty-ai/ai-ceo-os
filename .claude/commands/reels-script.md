> **트리거**: "AI 릴스 만들자", "릴스 써줘", "릴스 대본", "릴스 기획"
> **담당**: CCO (콘텐츠 책임자)
> **산출물 저장**: `outputs/{연도}/{월}/{일}/[릴스]_{주제}_{날짜}.md`

---

## 실행 순서 (무중단 파이프라인)

```
Phase 1: 주제 수신 → 후킹 패턴 선택
Phase 2: 대본 작성 (씬별 구성)
Phase 3: 캡션 + CTA + DM 템플릿 작성
Phase 4: 촬영 가이드 생성
Phase 5: 검수 → 저장
```

Phase 간 중간 보고 금지. 최종 결과물만 납품.

## STEP 1 — 주제 분석 & 후킹 패턴 선택

### 주제 검증 5단계 필터
1. **보편성**: 비전공자도 클릭할 주제인가?
2. **드라마 강도**: 30초 끝까지 볼 이유가 있는가?
3. **즉시 실용성**: 보고 나서 바로 써먹을 수 있는가?
4. **Before/After**: 변화가 명확히 보이는가?
5. **DM 리드 자산**: 댓글 받아서 줄 자료가 있는가?

### 6가지 후킹 패턴 (선택 기준 포함)

| 패턴 | 구조 | 언제 쓰나 |
|:---|:---|:---|
| **A. 금지/경고형** | "~하지 마세요" / "~쓰면 손해" | AI 도구 오남용 교정 주제 |
| **B. 숫자 충격형** | "N분 만에 / N시간 절약 / N%가 모르는" | 시간 절약·효율 주제 |
| **C. 사회적 낙오형** | "아직도 ~하시는 건 아니시죠?" | 구식 습관 교체 주제 |
| **D. 비밀 공개형** | "아무도 안 알려줘서 욕먹을 각오로" | 숨겨진 기능 소개 |
| **E. 인지 부조화형** | 상식 → 강부정 → 근거 → 대안 | 잘못된 상식 깨기 |
| **F. 리스트/저장형** | "상황별 딱 이거만 쓰면 끝" | 여러 도구·방법 비교 |

**후킹 문장 황금 공식**
```
[타겟 명시] + [금지/손실 프레임] + [암시적 보상]
→ "직장인이라면 이렇게 쓰지 마세요" (타겟+금지)
→ "아직도 GPT 이렇게 쓰고 계신 건 아니시죠?" (사회적 낙오)
→ "현직 APM 엔지니어가 욕먹을 각오로 다 공개합니다" (비밀공개+권위)
```

## STEP 2 — 대본 작성 (4가지 구조 중 선택)

### 구조 선택 기준

| 구조 | 적합 주제 |
|:---|:---|
| ① 문제→해결→증거 | AI 도구 기능 소개, 꿀팁 |
| ② 리스트형 | 여러 도구 비교, 상황별 추천 |
| ③ Before/After 비교 | 업무 방식 개선, 시간 절약 |
| ④ 스텝별 튜토리얼 | 설정법, 사용법 |

---

### 구조 ① 문제→해결→증거 (30초 기준)

```
[씬1 · 0~3초] 얼굴 등장 · 후킹 문장 말하기
→ 패턴 A/C/D/E 중 선택
→ 예: "직장인이라면 이거 모르고 Claude 쓰지 마세요"

[씬2 · 3~10초] 화면 레코딩 시작 · 고통 자극
→ 손실 회피 프레임
→ "매번 3분씩 — 한 달이면 [계산된 숫자] 그냥 버리는 겁니다"

[씬3 · 10~20초] 솔루션 공개 + 권위 레이어
→ "현직 엔지니어가 실제 쓰는 방법 공개합니다"
→ 화면에서 직접 시연 (클릭 동선 보여주기)

[씬4 · 20~27초] Before/After 또는 혜택 요약
→ 숫자 포함 ("3분 설정 → 반복 설명 0번")
→ "상위 0.1% [도구] 유저가 이미 쓰는 방법"

[씬5 · 27~30초] CTA — 댓글 키워드 + 보상 예고
→ "댓글에 '[키워드]' 남기시면 [자료] 바로 보내드립니다"
```

---

### 구조 ② 리스트형 (저장 유도 최강)

```
[씬1 · 0~3초] 얼굴 · 후킹
→ "상황별 딱 이거만 쓰면 끝 — [N]가지 공개합니다"

[씬2~4 · 3~25초] 리스트 공개
→ 상황 1 → 도구/방법 A (화면 시연)
→ 상황 2 → 도구/방법 B
→ 상황 3 → 도구/방법 C (가장 임팩트 있는 것 마지막)

[씬5 · 25~30초] 저장 CTA
→ "지금 저장해두세요 — 나중에 못 찾습니다"
→ "더 자세한 가이드는 댓글에 '[키워드]' 남겨주시면 드립니다"
```

---

### 구조 ③ Before/After 비교

```
[씬1 · 0~3초] 얼굴 · 대조 후킹
→ "3개월 전 나 vs 지금 나 — 이 하나로 달라졌습니다"

[씬2 · 3~12초] Before 묘사 (공감)
→ 구체적 고통 + 시간/비용 수치

[씬3 · 12~22초] After 공개 (시연)
→ 화면으로 직접 보여주기
→ 불규칙 숫자 사용 ("87분 → 7분")

[씬4 · 22~27초] 결과 요약 + 희소성
→ "이거 쓰는 사람 아직 드뭅니다"

[씬5 · 27~30초] CTA
```

---

### 구조 ④ 스텝별 튜토리얼

```
[씬1 · 0~3초] 얼굴 · 결과 먼저 보여주기
→ "이렇게 됩니다 — 방법 바로 알려드릴게요"

[씬2~4 · 3~25초] Step 1 → Step 2 → Step 3
→ 각 단계 5~7초, 화면 클릭 동선 자막 포함
→ 단계 수: 최대 3단계 (복잡하면 이탈)

[씬5 · 25~30초] 완성 결과 재확인 + CTA
→ "복붙 가능한 [자료] 드릴게요 — 댓글에 '[키워드]' 남겨주세요"
```

## STEP 3 — 문장 품질 기준

### 후킹 문장 레벨 시스템

| 레벨 | 구조 | 예시 |
|:---|:---|:---|
| Lv1 (약함) | 단순 손실 | "이거 모르면 손해" |
| Lv2 | 정량화 | "한 달 7시간 30분 버리는 겁니다" |
| Lv3 | 비교 손실 | "아직도 매번 처음부터 설명하시는 건 아니시죠?" |
| Lv4 | 사회적 낙오 | "직장인 95%가 이 기능 모르고 씁니다" |
| **Lv5 (최강)** | 3중 압박 | 타겟 명시 + 손실 + 사회적 고립 동시 |

→ **Lv3 이상 사용 원칙**

### 숫자 신뢰도 원칙
- 둥근 숫자 금지 → 불규칙 숫자 사용
  - ❌ "3시간" → ✅ "2시간 47분"
  - ❌ "100명" → ✅ "한국인 93%"
  - ❌ "많은 분들이" → ✅ "직장인 10명 중 9명이"

### 감정 롤러코스터 설계 (필수)
```
두려움/손실 (씬1~2) → 안도/해결 (씬3) → 성취/희소성 (씬4) → 행동 (씬5)
```

### 7초 규칙
- 매 7초마다 새로운 정보 또는 반전 배치 (주의력 리셋)

---

## STEP 4 — 캡션 작성 공식

```
[1~2줄] 손실 프레임 또는 충격 수치 (후킹과 동일 톤)
[3~5줄] 공감 + 해결책 요약 (릴스 내용 핵심)
[6~7줄] 댓글 CTA (키워드 + 보상 명시)
[해시태그] 5~8개 (AI관련 + 직장인 + 꿀팁 계열)
```

**캡션 금지 사항**
- "안녕하세요", "오늘은~" 류 인사 서론 금지
- CTA 2개 이상 금지 (분산 → 전환율 하락)
- "좋아요·저장 부탁드려요" 직접 요청 금지

---

## STEP 5 — CTA 설계 원칙

### If-Now-Then 구조
```
If (조건): "댓글에 '[키워드]' 남기시면"
Now (행동): "[AI]가 처리한 [자료]"
Then (결과): "지금 바로 보내드립니다"
```

### 댓글 키워드 선정 기준
- 3글자 이하 (AI, 템플릿, 자동화, 프롬프트)
- 타이핑 쉬운 단어
- 릴스 주제와 직접 연결

### DM 자료 항상 준비
- 복붙 가능한 프롬프트 or 템플릿
- 설정 방법 스크린샷 or 노션 링크
- 단계별 가이드 (3단계 이내)

---

## STEP 6 — 촬영 가이드 (산출물에 포함)

```
📹 촬영 방식
- 씬1 (0~3초): 얼굴 클로즈업 · 세로 · 밝은 배경
- 씬2~ : 화면 레코딩 (앱/웹 화면 직접 캡처)
- 배경음악: 볼륨 20~30% (목소리 > 음악)

✂️ 편집 포인트 (캡컷 기준)
- 2~3초마다 컷 전환
- 자막: 굵은 흰색 + 검정 테두리 · 핵심 키워드 강조
- 핵심 수치 → 크기 크게 + 색상 강조
- 전체 길이: 30초 이하 (초과 시 컷 편집)

⏰ 업로드 최적 시간
- 인스타그램: 18~21시 · 목/금/토 우선
```
