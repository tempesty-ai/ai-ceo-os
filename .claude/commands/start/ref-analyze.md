# 스킬: 레퍼런스 릴스 분석 → 대본 생성

> **트리거**: "레퍼런스 분석해줘 [URL]", "이거 참고해서 만들어줘 [URL]", "릴스 URL 줄게"
> **담당**: CCO
> **산출물**: `outputs/{연도}/{월}/{일}/[릴스]_{주제}_ref분석.md`

---

## 실행 파이프라인 (무중단)

```
Phase 1: Playwright로 URL 열기 → 스크린샷 + 구조 분석
Phase 2: 후킹/구조/CTA 해체 분석
Phase 3: 우리 채널 버전으로 대본 생성 (reels-script.md 기준)
Phase 4: 캡션 + DM 템플릿 작성
Phase 5: 검수 → 저장
```

---

## Phase 1 — 영상 분석 항목

Playwright로 URL 열고 다음 항목 파악:

| 항목 | 분석 내용 |
|:---|:---|
| 첫 프레임 | 얼굴 유무 / 텍스트 오버레이 / 색상 |
| 후킹 문장 | 첫 1~3초 텍스트 or 자막 |
| 구조 유형 | 문제→해결 / 리스트 / Before/After / 튜토리얼 |
| 후킹 패턴 | A~F 중 어느 패턴 |
| CTA | 댓글 키워드 / 저장 / 팔로우 |
| 영상 길이 | 초 |
| 자막 스타일 | 굵기 / 색상 / 강조 방식 |

---

## Phase 2 — 해체 분석 출력 형식

```
## 원본 분석
- 계정: @{계정명}
- URL: {URL}
- 후킹 패턴: {패턴명}
- 구조: {구조 유형}
- 후킹 문장: "{원문}"
- CTA: "{원문}"
- 참고 포인트: {우리가 가져갈 것}
- 버릴 것: {우리 포지셔닝과 맞지 않는 것}
```

---

## Phase 3 — 우리 채널 버전 대본 생성

`skills/reels-script.md` 기준 그대로 적용:
- 포지셔닝: 현직 APM 엔지니어 · 실전 AI 활용
- 타겟: AI에 관심 있는 직장인
- 얼굴: 씬1만, 이후 화면 레코딩
- 길이: 30초
- 주제: AI+인사이트 (기본값)

**원본 구조는 가져오되, 문장은 100% 우리 포지셔닝으로 재작성**

---

## 검수 기준

reels-script.md 검수 기준 동일 적용 (Q1~Q9 전항목)
