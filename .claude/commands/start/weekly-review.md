# /start:weekly-review — 금요일 주간 성과 리뷰

> 매주 금요일 실행. 이번 주 KPI 체크 → 회고 → 다음 주 방향 힌트.

## 실행 전 준비
- 성과 데이터가 있으면 CEO가 입력 (릴스별 조회수, 저장 수, 공유 수, 팔로우 유입 수)
- 데이터 없으면 KPI 체크 스킵 → Phase 2(회고) + Phase 3(방향 힌트)만 진행

## 실행 순서

### Phase 1: KPI 체크
1. 게시물 3개 성과 수집 (CEO 입력값 기준)
2. KPI 기준선 대비 비교:
   - 저장률 2%+ / 완료율 70%+ / 팔로우 전환율 0.5%+ / 공유율 1%+
3. 콘텐츠 축별 성과 비교: AI+인사이트 vs AI+육아 — 어느 쪽이 더 반응 좋았는가

### Phase 2: 회고
4. WIN 패턴 식별 → `tasks/lessons.md`에 "WIN:" 태그로 기록
5. 저성과 원인 분석 (증상 말고 근본 원인)
6. `tasks/lessons.md` 업데이트

### Phase 3: 다음 주 방향 힌트
7. 상위 성과 포맷 → 다음 주 `/start:weekly-start`에 반영할 방향 1줄
8. 버려야 할 것 1가지 (저성과 포맷/주제)
9. `tasks/session_handoff.md` 업데이트

## 산출물
| 항목 | 내용 |
|:---|:---|
| 이번 주 성과 요약 | 게시물 3개 KPI 테이블 |
| AI+인사이트 vs AI+육아 | 반응률 비교 |
| WIN 패턴 | 재사용할 것 |
| 다음 주 방향 힌트 | 1줄 |
