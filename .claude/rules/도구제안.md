# 도구 활용 제안 규칙

> CEO가 업무를 지시하면, 5가지 도구 중 활용 가능한 것을 자동으로 판단.
> 해당되면 **작업 시작 전에 CEO에게 먼저 제안**합니다.

## 5가지 도구

### 1. Plan Mode (플랜모드)
**제안 조건**: 새 기능 / 3개 이상 파일 / 여러 접근법 가능 / 모호한 요구사항
**문구**: "대표님, 이 작업은 [이유]라서 플랜모드로 설계안 먼저 잡고 진행하는 게 안전합니다."

### 2. Playwright (브라우저 자동화)
**제안 조건**: 경쟁사/레퍼런스 사이트 확인 / 우리 사이트 시각적 점검 / 웹 데이터 수집
**문구**: "대표님, [사이트]를 Playwright로 직접 열어서 캡처해드릴 수 있습니다."

### 3. Notion 연동
**제안 조건**: 결과물 노션 정리 / 노션 DB 참고 / "팀과 공유" "문서화" 키워드
**문구**: "대표님, 이 결과물을 노션에 정리해드릴 수 있습니다."

### 4. Worktree (격리 실험)
**제안 조건**: 실험적 변경 / A/B 비교 / "한번 해봐" "테스트로" 키워드
**문구**: "대표님, 이 작업은 워크트리에서 먼저 해보는 게 안전합니다."

### 5. Context7 (라이브러리 문서)
**제안 조건**: 특정 라이브러리/프레임워크 최신 사용법 / 새 패키지 도입
**문구**: "대표님, [라이브러리]의 최신 문서를 Context7로 조회하고 진행하겠습니다."

## 실행 규칙
1. **자동 판단, 사전 제안**: 매 작업 지시 시 5가지를 빠르게 스캔
2. **강제 아님**: CEO가 "그냥 해" 또는 "필요 없어"라고 하면 즉시 제안 없이 진행
3. **복수 해당 가능**: 여러 도구가 동시에 해당되면 묶어서 제안
4. **과도한 제안 금지**: 단순 작업에는 제안하지 않음
5. **제안 형식**: 작업 응답 맨 앞에 1~2문장으로 짧게
