# CEO/ — CEO 개인 공간

> CEO만의 작업 공간입니다. 김이사가 참고하지만 함부로 수정하지 않습니다.
> 이 폴더는 기본적으로 git 추적에서 제외됩니다 (`.gitignore`).
> 본인 데이터는 안전하게 로컬에서만 관리됩니다.

## 폴더 구조

```
CEO/
├── 아이디어.md       ← 떠오르는 사업 아이디어 메모
├── 일정_마감.md      ← 중요한 마감일, 미팅
├── 회고.md           ← 주간/월간 회고
├── 목표.md           ← 분기/연간 목표
└── 멘탈모델.md       ← CEO의 사고 방식, 신념
```

## 사용법

### 김이사가 자동으로 참조하는 시점

- **일정_마감.md**: 매 세션 시작 시 D-7 이내 마감 자동 감지 → 선제 알림
- **목표.md**: 모든 전략 판단 시 정합성 체크
- **멘탈모델.md**: 콘텐츠 톤, 의사결정 스타일 적용
- **회고.md**: 패턴 학습 (CEO가 자주 후회하는 결정 등)
- **아이디어.md**: 신상품/신사업 검토 시 참조

### CEO가 직접 사용하는 방법

- **아이디어가 떠오르면**: `아이디어.md`에 1줄 메모. 김이사에게 "이거 검토해줘" 하면 됨
- **회고하고 싶을 때**: "주간 회고 도와줘" → 김이사가 데이터 + 회고.md 참고하여 작성
- **목표 변경 시**: `목표.md` 직접 수정 → 김이사가 다음 작업부터 반영

## 보안

기본 starter 파일(`목표.md`, `일정_마감.md`, `아이디어.md`)은 빈 슬롯 상태로 git에 추적됩니다.

**개인 데이터를 채운 후에는** 로컬 `.gitignore`에 추가해서 본인 데이터가 GitHub에 푸시되지 않도록 보호하세요:

```
# .gitignore에 추가
CEO/목표.md
CEO/일정_마감.md
CEO/아이디어.md
```

또는 git의 `--skip-worktree` 기능을 사용해도 됩니다:

```bash
git update-index --skip-worktree CEO/목표.md
git update-index --skip-worktree CEO/일정_마감.md
git update-index --skip-worktree CEO/아이디어.md
```
