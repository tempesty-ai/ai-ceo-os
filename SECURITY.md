## 보안 취약점 신고

이 레포에서 보안 취약점을 발견하셨다면, **공개 Issue로 등록하지 마시고** 아래 절차를 따라주세요:

### 1. 비공개 신고

- GitHub Security Advisory 기능 사용 (권장)
- 또는 프로젝트 관리자 이메일로 직접 연락

### 2. 신고 시 포함할 정보

- 취약점 유형 (예: 민감정보 노출, 명령어 주입, 권한 상승)
- 영향 범위 (어떤 파일/스킬/스크립트)
- 재현 절차
- 영향도 추정 (저/중/고/치명)
- 가능하면 수정 제안

### 3. 대응 프로세스

| 단계 | 기한 |
|:---|:---|
| 접수 확인 | 48시간 내 |
| 검증 및 분류 | 7일 내 |
| 패치 개발 | 취약점 심각도에 따름 |
| 공개 공지 | 패치 배포 후 |

## 주요 보안 고려 사항

### API 키 / 시크릿 관리
- `.env` 파일은 절대 커밋하지 마세요. `.gitignore`에 포함되어 있습니다
- `execution/sanitize_for_public.py` 스크립트로 공개 전 점검
- Claude API 키, Meta/Google 토큰, Notion 토큰 등은 환경변수로만

### 예제 데이터
- `examples/` 하위 페르소나는 **전부 허구**
- 실제 고객/클라이언트 이름, 연락처, 매출 데이터 포함 금지

### 스크립트 실행
- `scripts/` 및 `execution/` 스크립트는 사용자 환경에서 실행됩니다
- 실행 전 내용을 확인하는 것이 사용자 책임입니다
- 신뢰할 수 없는 출처의 PR은 실행 전 리뷰 필수

### Claude Code 권한
- `.claude/settings.json`의 `permissions.deny`에 위험 명령(예: `rm -rf /`, `sudo`)이 차단되어 있습니다
- 수정할 때는 의도를 명확히 하고 PR 리뷰를 거치세요

---

## 책임의 한계

이 레포는 **"AS IS"**로 제공됩니다 (MIT 라이선스). 사용자는 본인 환경에서 스크립트를 실행하기 전 반드시 검토해야 하며, 발생하는 문제에 대한 책임은 사용자 본인에게 있습니다.
