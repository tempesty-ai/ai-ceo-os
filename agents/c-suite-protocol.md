# C-Suite 운영 프로토콜 (3-Tier)

> 9명의 임원을 매번 전부 부르지 않습니다. 안건의 무게에 따라 3단계로 운영합니다.

---

## 3-Tier 운영

| Tier | 소집 | 발동 조건 | 비중 |
|:---|:---|:---|:---:|
| **Tier 1** | **김이사 단독** (내장 렌즈+참모진) | 일상적 판단, 단순 요청 | **90%** |
| **Tier 2** | 관련 C-level **2~3명 병렬** | 광고 변경, 콘텐츠 전략, 시스템 변경, 운영 조정 | **9%** |
| **Tier 3** | **8명 전원** + 상호 반박 라운드 | 가격 전략, 신상품 런칭, 분기 리뷰, "전체 검토해" | **1%** |

---

## Tier 1 — 김이사 단독

- 김이사가 4명의 참모진(회의론자/실용주의자/비전가/CEO대리인)을 내면 적용
- 8개 C-Suite "렌즈"를 머릿속으로 빠르게 통과
- 30초~5분 안에 결론

**언제 사용:**
- 단순 작업 요청 ("릴스 대본 1편 써줘")
- 기존 패턴 반복 작업
- 정보 검색/요약

---

## Tier 2 — 관련 임원 2~3명 병렬 소집

**언제 사용:**
- 광고 캠페인 변경/증액
- 콘텐츠 전략 재설계
- 신규 워크플로우 구축
- 시스템/인프라 변경
- 운영 프로세스 변경

**소집 조합 예시:**

| 안건 | 소집 임원 |
|:---|:---|
| 광고 ROAS 개선 | CMO + CDO + CFO |
| 콘텐츠 전환율 개선 | CCO + CMO + CDO |
| 강의 완강률 개선 | CPO + CXO + CDO |
| 신규 자동화 도입 | CTO + COO + CFO |
| 커뮤니티 활성화 | CXO + CCO + COO |
| 가격 정책 검토 | CMO + CFO + CPO |

**프로세스:**
1. 김이사가 안건 분석 → 관련 2~3명 선정
2. 각 임원에게 **독립 분석** 요청 (서로 안 보임)
3. 결과 수집 → 김이사 통합 → 결론
4. CEO에게 보고 (의견 충돌 있으면 그것도 함께 보고)

---

## Tier 3 — 8명 전원 + 반박 라운드

**언제 사용:**
- 가격 전략 결정
- 신상품/신서비스 런칭
- 분기 리뷰
- 전사 전략 변경
- CEO가 "전체 검토해" 또는 "임원회의 소집" 지시

**프로세스:**
1. **Round 1 (R1)**: 8명 전원이 독립적으로 안건 분석. 서로 안 보임.
2. **Round 2 (R2 — 반박 라운드)**: R1 결과를 모두에게 공유 → 각자 다른 임원의 의견에 반박/동의
3. **결론**: 김이사가 R1 + R2를 종합해 최종 결론 도출
4. **소수 의견 보존**: 다수와 다른 의견은 별도 기록 → 검증일 설정 → `tasks/predictions.md`에 등재

---

## 임원 충돌 시 처리

- **데이터 우선**: 의견이 충돌하면 데이터와 근거가 강한 쪽 채택
- **우선순위 매트릭스**: 매출 직결 > 고객 접점 > 브랜딩 > 내부 운영
- **소수 의견 폐기 금지**: 채택되지 않은 의견도 `predictions.md`에 검증일과 함께 기록 → 나중에 옳았는지 검증

---

## 검수 (자동)

모든 Tier 2/3 회의 결과는 다음을 포함해야 합니다:

- [ ] 각 임원의 핵심 입장 1줄 요약
- [ ] 합의 사항 / 충돌 사항 분리
- [ ] 김이사의 최종 결론 + 확신도(%)
- [ ] CEO 액션 아이템 (담당자, 기한)
- [ ] 소수 의견 기록 (있으면)

---

## 운영 팁

- **Tier 1을 90%로 유지하라**. Tier 2/3을 남발하면 의사결정 속도가 느려집니다.
- **Tier 3은 한 달에 1~2번 정도가 적정**. 너무 자주 하면 임원들이 피로해집니다(=AI 컨텍스트 비용 폭증).
- **소수 의견은 반드시 보존**. 대부분의 혁신은 소수 의견에서 나옵니다.
