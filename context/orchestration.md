# 오케스트레이션 규칙

> AI 에이전트들이 협업하는 방식.

## 3-Phase 파이프라인

```
Phase 1: 준비 — 관련 자료 확인, 맥락 파악 (CLAUDE.md, lessons.md, channel-rules.md)
  ↓
Phase 2: 실행 — 초안 작성
  ↓
Phase 3: 검증 — 다른 임원 관점에서 체크 (품질 기준, 톤, 금지 사항)
  ↓
최종 납품
```

## 에이전트 운영

- CSO 김이사가 전체를 조율
- `agents/_shared/routing-map.yaml`에 따라 안건별 임원 자동 소집
- 단순 판단은 김이사 단독 (Tier 1)
- 2~3명 렌즈 적용 (Tier 2)
- 전사 안건은 전임원 소집 (Tier 3)
- 의견 충돌 시 데이터·근거 강한 쪽 채택

## Tier 기준

| Tier | 설명 | 예시 |
|:---|:---|:---|
| Tier 1 | 김이사 단독 or 담당 1명 | 기술 자동화, 운영 프로세스 |
| Tier 2 | 2~3명 협업 | 릴스 대본, 데이터 분석, 커뮤니티 |
| Tier 3 | 전임원 소집 | 가격/런칭, 전사 전략 전환 |

## 품질 기준
- 숫자는 반드시 데이터 출처 명시 (Zero-Inference)
- 액션 아이템에 기한 포함
- 요약 먼저, 상세 나중
- CEO 톤앤매너 준수 (`context/ceo-persona.md`)
- 금지 행동 준수 (`CLAUDE.md`)
