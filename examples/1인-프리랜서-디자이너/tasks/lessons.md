## 기록

### 2026-03-15 — [C] 견적서 없이 단가만 DM으로 보냈다가 이탈 70%

- **상황**: 인스타 DM으로 "로고 얼마예요?" 물은 3명에게 "180만원부터 시작이에요"라고 답했더니 3명 모두 "감사합니다" 후 잠수.
- **잘못된 판단**: "빠른 답변이 좋다"고 생각해서 상담 프로세스 생략.
- **5 Whys**:
  1. 왜 이탈했나? → 숫자만 보고 "비싸다" 판단
  2. 왜 비싸 보였나? → **무엇을 받는지 모르는데** 가격만 보임 = 비교 기준이 없음
  3. 왜 기준을 안 줬나? → 패키지 구성 PDF를 첨부 안 함
  4. 왜 첨부 안 했나? → 매번 보낼 PDF가 없었음 (수동으로 그때그때 답변)
  5. 왜 PDF가 없었나? → 표준 패키지 문서가 없음

- **교훈**: 가격 질문은 **"가격 + 뭘 받는지"**가 한 쌍이어야 한다. 한 쌍이 아니면 고객 뇌는 "비싸다"로만 해석.

- **재발 방지 규칙**:
  1. **표준 패키지 PDF** 작성 (기본/플러스/리테이너 3종, 각 1페이지)
  2. DM 답변 템플릿에 PDF 자동 첨부
  3. 가격 답변 시 반드시 "+ 30분 무료 상담" 묶어서 제안
  4. CMO 서브에이전트에게 "DM 답변 초안" 의뢰 시 위 규칙 자동 적용

- **클러스터**: C

### 2026-03-22 — [A] 같은 로고 수정 4번 → 계약 범위 모호의 결과

- **상황**: 한 고객이 로고 시안을 4번 수정 요청. 계약서에 "수정 3회"로 명시했는데 고객이 "3차 수정 후 다시 수정 요청"을 4번째로 하지 않았다고 우김.
- **잘못된 판단**: 계약서 문구를 "수정 3회"로만 적고 "수정 3회 = 시안 3번 수정"이라는 명시 없음.
- **5 Whys**:
  1. 왜 분쟁? → 해석 차이
  2. 왜 해석 차이? → 계약서 문구가 추상적
  3. 왜 추상적? → 표준 계약서 템플릿을 대충 복붙
  4. 왜 대충? → 프리랜서 3년차인데 **계약서 템플릿을 한 번도 점검 안 함**
  5. 왜 점검 안 했나? → 분쟁이 없었어서 필요성 못 느낌

- **교훈**: 분쟁이 없었던 건 운이지 시스템이 아님. 템플릿은 **분쟁 전에** 점검해야 한다.

- **재발 방지 규칙**:
  1. 표준 계약서 재작성: "수정 3회 = 시안(안) 3개 제시 + 각 시안 내 수정 요청 각 2회까지"
  2. 계약서 서명 전 **"수정 정책 1분 설명"** 구두 안내 (오해 사전 차단)
  3. 4회 초과 수정 시 **건당 15만원 추가** 명시
  4. 모든 표준 문서(계약서/견적서/패키지 PDF)는 분기 1회 점검 (3개월마다 lessons.md에 "템플릿 리뷰" 일정 등록)

- **클러스터**: A

### WIN: 2026-03-28 — 포트폴리오 포스트 "문제 → 접근 → 결과" 구조 확립

- **상황**: 3월 동안 5개 포트폴리오 포스트 발행. 그 중 2개는 "완성 이미지만", 3개는 "문제→접근→결과" 구조.
- **결과**: 구조화 3개의 평균 저장률이 구조 없는 2개 대비 **3.2배** 높음 (데이터: `knowledge/channel-data/instagram_2026MMDD.json`).
- **배운 것**: 디자이너 본능("결과 예쁘면 됐지")과 고객 뇌("이 사람이 내 문제도 풀어줄까?")는 다르다. 고객은 **"내 경우에도 적용 가능한가?"**를 판단하고 싶어한다. 스토리 구조가 그 판단을 도와줌.
- **재사용 규칙**: 모든 포트폴리오 포스트는 이 3단 구조 강제. CCO에게 포스트 초안 요청 시 **"3단 구조 준수"를 프롬프트에 포함**.
- **태그**: `WIN:` + 클러스터 A(프로세스 확립)
