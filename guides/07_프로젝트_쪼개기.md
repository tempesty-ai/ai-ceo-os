# Guide 07 — 프로젝트 쪼개기 (AI 사업부 만들기)

> **선행 조건**: `/setup` 을 완료해서 본사(ai-ceo-os 루트)가 세워져 있어야 합니다.
> **소요 시간**: 첫 사업부 30~40분, 두 번째부터 10분.
> **결과물**: `projects/{{내 사업부}}/` 폴더 1개 (하네스 5종 포함)

---

## 이 가이드가 다루는 것

1. 왜 본사 하나로는 부족한가
2. 하네스(Harness) 5종 — 사업부 안에 뭘 넣나
3. 쪼개기 3대 원칙
4. 첫 사업부 만들기 Step-by-Step
5. 두 번째 사업부 — 복제로 10분

---

## 1. 왜 쪼개야 하나

`/setup` 을 돌리면 본사 `CLAUDE.md` 가 완성됩니다. 하지만 같은 AI 에게:

> "이번 주 인스타 콘텐츠 3개 기획해줘"

라고 물으면 답이 **범용적**으로 나옵니다. 내 업종/톤/채널 반영이 부족합니다.

**이유**: 본사 `CLAUDE.md` 는 회사 전체 헌법이라, "인스타 채널 규칙" 같은 세부 사항을 넣으면 파일이 비대해져 AI 가 중요한 걸 놓칩니다. 그래서 **세부 맥락은 사업부 폴더로 내려보냅니다.**

### 본사 vs 사업부 역할 분담

| 구분 | 본사 | 사업부 |
|:---|:---|:---|
| 역할 | 전사 헌법 | 실전 작업장 |
| CLAUDE.md 길이 | 길어도 OK | 30~50줄 |
| 담당 | C-Suite 9명 전체 | 이 팀 담당 1~3명 |
| 질문 예시 | "우리 회사 방향은?" | "이번 주 광고 예산 배분해줘" |

## 2. 하네스 5종 세트

> "말을 탈 때 고삐·안장·등자가 필요한 것처럼, AI 를 일 시키려면 장치 세트가 필요합니다."

```
projects/marketing/
├── CLAUDE.md       ← ① 사업부 헌법 (미션/KPI/담당)
├── agents/         ← ② 담당 임원 (C-Suite에서 @참조)
├── skills/         ← ③ 반복 업무 SOP
├── context/        ← ④ 채널/환경 규칙
└── rules/          ← ⑤ 자동 적용 금지/품질 룰
```

| # | 하네스 | 없으면 |
|:---:|:---|:---|
| ① `CLAUDE.md` | 팀 방향 없음 |
| ② `agents/` | "누가 할 일인데요?" |
| ③ `skills/` | 매번 처음부터 설명 |
| ④ `context/` | 결과 범용화 |
| ⑤ `rules/` | 사고 반복 |

## 3. 쪼개기 3대 원칙

### 원칙 1. 업무 카테고리로 쪼갠다 (채널 X)

❌ `insta/ · youtube/ · blog/` — "릴스 기획" 스킬을 3개 폴더에 복붙해야 함
✅ `marketing/ · dev/ · operations/ · customer/` — 업무 로직 재사용

채널별 차이는 `context/channel-rules.md` 에서 구분.

### 원칙 2. 처음엔 2~3개로 시작한다

한 번에 6개 만들면 다 무너집니다. 매출 직결되는 1개부터.

### 원칙 3. 하나 잘 만들고 복제한다

첫 사업부 30~40분 → 두 번째 10분 → 세 번째 5분 → 네 번째부턴 무료.
**첫 하나가 템플릿이 됩니다.** 대충 만들면 안 됩니다.

## 4. 첫 사업부 만들기 — Step by Step

### Step 0. 내 사업 쪼개기 (5분)

노트에 적으세요:

1. 나는 어떤 사업인가?
2. 매주 반복되는 업무 3~5개
3. 그걸 업무 카테고리 2~3개로 묶으면?
4. 그중 **매출 직결 1순위**는? ← 오늘은 이거 하나만

### 업종별 예시

| 업종 | 1순위 사업부 | 담당 |
|:---|:---|:---|
| 네일샵 | marketing (예약 유입) | CCO + CMO |
| 코칭 | sales (상담 → 전환) | CCO + CPO |
| 쇼핑몰 | marketing (신상 릴스) | CCO + CMO |
| 1인 개발 | dev (MVP 출시) | CTO |

### Step 1. 템플릿 복제 (1분)

**직접 `mkdir` 하지 마세요.** Claude 에게 시키는 게 빠릅니다.

```
"projects/_template 을 복제해서 projects/marketing 사업부를 만들어줘."
```

### Step 2. CLAUDE.md 채우기 (10분)

```
"projects/marketing/CLAUDE.md 의 {{슬롯}} 을 내 사업(네일샵 예약 유입)에
 맞춰 채워줘. KPI 는 주간 릴스 발행 수, DM 문의 수, 링크 클릭률 3개."
```

### Step 3. agents/ — 담당 임원 선언 (5분)

```
"projects/marketing/agents/README.md 에 주 담당 CCO, 부 담당 CMO 로
 선언하고, 본사 경로 @../../../agents/cco/README.md 형식으로 연결해줘."
```

> **핵심**: 복사하지 말고 **@참조** 로 연결. 본사에서 업데이트하면 자동 반영.

### Step 4. skills/ — 첫 스킬 1개 (10분)

```
"projects/marketing/skills/reels-script.md 를 만들어줘.
 트리거는 '릴스 대본 써줘', 3-Phase 파이프라인 + 3버전 생산 원칙,
 검증 체크리스트 포함."
```

> **3버전 원칙**: 대본/캡션은 항상 3개. AI 첫 답이 최선이 아닙니다.

### Step 5. context/channel-rules.md (5분)

```
"projects/marketing/context/channel-rules.md 에 인스타 운영 규칙을
 넣어줘. 릴스 주 4개, 캐러셀 주 2개, 발행 시간 화/목/토 18:00."
```

### Step 6. rules/금지행동.md (5분)

```
"projects/marketing/rules/금지행동.md 에 4개 룰 넣어줘:
 경쟁사 비방 금지, 할인 언급 금지, 효과 보장 금지, 미검증 후기 금지."
```

### Step 7. 완성 체크

```bash
ls projects/marketing/
# CLAUDE.md  agents  skills  context  rules  ← 5개 다 있으면 OK
```

## 5. 두 번째 사업부 — 10분 안에

```
"projects/marketing 구조를 그대로 복제해서 projects/customer 사업부를 만들어줘.
 담당 임원은 CXO + CCO 로 바꾸고,
 CLAUDE.md 미션은 '기존 고객 재구매/이탈 방지' 로,
 skills 는 dm-reply.md 하나만 넣어줘."
```

복제 후 달라져야 할 것만 치환됩니다.

## 6. 검증 — 같은 질문, 다른 답

Before (본사에서):

```
"이번 주 릴스 3개 기획해줘"
→ 범용 답변, 내 톤/채널 미반영
```

After (`cd projects/marketing` 후):

```
"이번 주 릴스 3개 기획해줘"
→ CCO 자동 소집 + channel-rules.md 로드 + 금지행동 자동 체크
→ 3버전 대본 + 캡션 + CTA + 발행 시간까지
```

이 체감 차이가 나면 성공입니다.
