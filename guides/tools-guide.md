# 도구 활용 가이드

> 클로드코드 + AI CEO OS를 쓰면서 강력해지는 5가지 도구.
> 김이사가 작업 시작 전에 자동으로 제안하는 도구들입니다.

---

## 1. Plan Mode (플랜모드)

### 언제?
- 새 기능 개발, 3개 이상 파일 건드리는 작업
- 여러 접근법 가능한 의사결정
- 요구사항이 모호한 작업

### 사용법
```
Shift + Tab을 두 번 누르면 Plan Mode 진입
→ AI가 코드를 쓰지 않고 "계획서"만 만듦
→ 검토 후 승인하면 그때 실행
```

### 효과
실수로 잘못된 코드를 작성하기 전에 방향을 잡을 수 있습니다.

---

## 2. Playwright (브라우저 자동화)

### 언제?
- 경쟁사 사이트 직접 확인
- 우리 랜딩페이지 시각적 점검
- 웹에서 데이터 수집

### 사용법
```
"경쟁사 ㅇㅇ 사이트 들어가서 가격 페이지 캡처해줘"
→ AI가 브라우저를 직접 열어서 화면 캡처 + 분석
```

### 효과
직접 사이트에 들어가는 시간 0초. AI가 대신 봐주고 요약해줍니다.

---

## 3. Notion 연동

### 언제?
- 작업 결과물을 노션에 정리
- 노션 DB 데이터 참고
- 회의록/문서화 작업

### 사용법
```
"이 분석 결과를 노션에 새 페이지로 만들어줘"
→ AI가 노션 API로 페이지 자동 생성
```

### 효과
보고서를 직접 옮겨 적을 필요가 없습니다.

---

## 4. Worktree (워크트리)

### 언제?
- 실험적 코드 변경 (실패해도 되돌릴 수 있어야 함)
- A/B 비교 작업
- 본 작업에 영향 주지 않고 별도로 시도

### 사용법
```
"워크트리에서 새 디자인 시안 시도해봐"
→ 격리된 환경에서 작업, 마음에 들면 합치고 안 들면 버림
```

### 효과
"이거 잘못 건드리면 큰일 나는데..." 부담 없이 실험 가능.

---

## 5. Context7 (라이브러리 문서 조회)

### 언제?
- 라이브러리/프레임워크 최신 사용법 확인
- 새 패키지 도입 검토
- 에러가 라이브러리 사용법 관련일 때

### 사용법
```
"Next.js 15의 새 라우팅 방식 확인하고 적용해줘"
→ AI가 Context7로 최신 공식 문서 조회 후 적용
```

### 효과
AI 학습 데이터가 오래되어 잘못된 코드를 쓰는 문제를 방지합니다.

---

## 김이사가 자동 제안하는 규칙

이 5가지는 김이사가 작업 지시를 받을 때 **자동으로 판단**해서 먼저 제안합니다. 예를 들어:

- "새 기능 만들어줘" → "Plan Mode 들어갈까요?"
- "경쟁사 어떻게 생겼어?" → "Playwright로 캡처해드릴까요?"
- "노션에 정리해줘" → "Notion 연동으로 자동 생성하겠습니다"
- "실험적으로 해봐" → "워크트리에서 진행할까요?"
- "이 라이브러리 어떻게 써?" → "Context7로 최신 문서 확인하겠습니다"

CEO가 "필요 없어"라고 하면 그냥 진행합니다. 강제 아닙니다.
