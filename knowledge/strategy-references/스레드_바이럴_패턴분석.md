# 스레드 바이럴 패턴 분석

> 수집 기간: ~2025.09.15 / 출처: 스레드 AI 계정 모음
> 활용: 콘텐츠 기획 시 후킹 공식·포맷 참고용

---

## 핵심 인사이트: 조회수별 공식 패턴

### A. 초고조회수 (30만~103만) — 공통 패턴

| 후킹 공식 | 예시 | 조회수 |
|:---|:---|:---:|
| "어차피 알려줘도 100명 중 97명 안 함" | AI 안 쓰면 돈 못 법니다 | 80만 |
| 페르소나 타겟 + 숫자 리스트 | "대학생들 열어보지 마세요" | 34만 |
| "절대 유출 금지" / "대기업 방식" | 마케팅팀 프롬프트 공개 | 47만 |
| 저장유도 + CTA 강화 | "저장 안하면 너만 손해" | 47만 |
| 상황별 도구 매칭 리스트 | "딱 하나만 유료 결제" | 10만 |

### B. 중간 조회수 (5만~20만) — 성공 요인

- **실전 노하우** + 구체적 수치 (시간 단축, 비용 절감)
- 무료 자료 증정 + 팔로우/댓글 CTA 조합
- 특정 직업군 타겟 ("직장 막내", "대학원생", "인사팀")
- 스토리텔링 ("할머니가 Gemini 얘기하는 걸 보고 정신이 번쩍")

### C. 저조회수 패턴 (1만~3만) — 피해야 할 것

- 단순 도구 나열 (차별화 없음)
- 너무 기술적인 설명 (업스케일링, ComfyUI 등)
- CTA 없이 정보만 전달

## 조회수 상위 후킹 문장 모음

```
- "AI 안 쓰면 올해도 돈 못 법니다 / 꼭 알려줘도 안쓰더라;"
- "어차피 알려줘도 100명 중 97명은 안함"
- "대학생들은 열어보지 마세요, 너무 유용할 수 있습니다"
- "절대 유출 금지로 관리하던 AI 프롬프트 템플릿을 공개합니다"
- "안쓰면 진짜 바보되는 ~"
- "저장 안하면 너만 손해임"
- "[직장 막내들 주목] 이거 알면 회사생활 쉽다"
- "대학원생이 알았으면 좋았을 정보들"
- "할머니가 Gemini 얘기하는 걸 보고 정신이 번쩍 들었다"
```

## 고조회수 콘텐츠 구조 분석

### 구조 ①: 상황별 도구 매칭 (10만~80만)
```
[후킹] "AI 안 쓰면 X"
↓
[번호 리스트] 1. 상황A → 도구A / 2. 상황B → 도구B ...
↓
[CTA] 팔로우 or 저장 유도
```

### 구조 ②: 프롬프트 공개형 (25만~47만)
```
[후킹] "~하는 법 알려줄게" or "유출 금지 자료 공개"
↓
[단계별 프롬프트] 1단계→2단계→3단계
↓
[CTA] "저장하고 써보자" / 댓글 유도
```

### 구조 ③: 무료 자료 증정형 (4만~10만)
```
[후킹] 결과물 이미지/영상 선공개
↓
[내용] 어떤 툴로 어떻게 만들었는지
↓
[CTA] "팔로우 + 댓글 → DM 발송"
```

### 구조 ④: 스토리텔링형 (10만~38만)
```
[충격/공감 장면] "할머니가 Gemini 얘기를..." / "10억 날리고 알게 된..."
↓
[인사이트 1~3개]
↓
[여운 있는 마무리] (CTA 없어도 됨)
```

## 우리 채널 적용 포인트

1. **후킹 공식 1순위**: "어차피 알려줘도 안 함" 프레임 → AI+인사이트 도구 리스트
2. **후킹 공식 2순위**: 페르소나 타겟 ("육아하는 부모가 몰랐던 AI 5가지")
3. **CTA 공식**: 저장 유도 or 팔로우+댓글 → DM 증정
4. **피해야 할 것**: 기술 용어 설명형, CTA 없는 단순 나열
5. **스토리 소재**: 현직 엔지니어 시점의 "정신 번쩍" 에피소드 발굴
