## 1. 요약 (TL;DR)

우리 시스템은 **"9명 C-Suite 에이전트 + Tier 프로토콜 + 한국어 경영 DSL"** 이라는 고유 강점이 있지만, 에이나우 가이드가 체계화한 **"Boris Cherny 워크플로우 + 병렬세션 + 자기학습 루프 + 플러그인/Hook 자동화"** 인프라 레이어가 **거의 비어 있다.**

- **현재 강점**: 전략 사고 프레임(Cross-Lens, MUST Disagree, Inversion), 사업 도메인 온톨로지(사업부/고객여정/KPI), 한국어 페르소나
- **치명적 갭 3건** (P0): ① Hook 시스템 0건 ② 자기학습 루프(lessons → CLAUDE.md 자동 반영) 구조적 미비 ③ 병렬세션/Worktree 가이드 부재
- **확신도**: 87% (PDF 142p 전체 스캔 + 현재 레포 구조 교차검증 완료)

**결론**: 우리는 "전략 OS"를 만들었고, 에이나우 가이드는 "생산성 OS"를 만들었다. **두 레이어는 충돌하지 않고 합쳐질 수 있다.** 아래 12개 디벨롭 항목 중 **P0 3개만 먼저 적용해도 수강생 첫 경험이 눈에 띄게 개선된다.**

## 4. 디벨롭 로드맵 (우선순위 + 구체 실행안)

### 🔴 P0 — 즉시 (이번 주 내)

#### P0-1. Hook 시스템 도입 (`자동검수.md`의 "100% 실행" 보장)

**문제**: 현재 `.claude/rules/자동검수.md`는 "모든 스킬 실행 후 자동 체크"라고 선언하지만, **실제로는 Claude가 80% 확률로만 따른다.** PDF Ch.5가 지적한 핵심 한계.

**해법**: `.claude/settings.json`에 Hook 3종 추가.

```json
"hooks": {
  "PostToolUse": [
    { "matcher": "Write|Edit",
      "hooks": [{"type":"command","command":"# outputs/ 경로 검증 스크립트"}] }
  ],
  "PreToolUse": [
    { "matcher": "Edit|Write",
      "hooks": [{"type":"command","command":"# .env, knowledge/ 보호"}] }
  ],
  "Stop": [
    { "hooks": [{"type":"command","command":"# 세션 종료 시 tasks/session_handoff.md 자동 업데이트"}] }
  ]
}
```

**기대효과**: 검수 누락 0%, 산출물 경로 오류 0%, 세션 간 인수인계 자동화.

#### P0-2. 자기학습 루프 자동화 (Boris 황금 규칙)

**문제**: 현재 `tasks/lessons.md` 는 있지만, "Claude가 실수 → 규칙화 → CLAUDE.md에 반영" 루프가 **수동**이다.

**해법**:
1. `.claude/rules/공통.md`에 명시적 6단계 루프 추가 ("작업 종료 전 반드시: 이번 세션에서 배운 것 1개 이상을 `tasks/lessons/{카테고리}.md`에 기록")
2. `tasks/lessons/` 폴더 카테고리화: `cso.md`, `cmo.md`, ... `오류패턴.md`
3. 월 1회 `lessons → CLAUDE.md 교훈 섹션` 승격 워크플로우 (`/lessons-review` 스킬 신설)

**기대효과**: 1개월 후 CLAUDE.md가 실전 교훈 20~30개 축적 → 에이전트 실수율 체감 급감.

#### P0-3. 병렬 세션 가이드 신설 (`guides/08_병렬세션.md`)

**문제**: Tier 2~3 동시 호출, 사업부 간 병렬 작업을 수강생이 어떻게 운영할지 가이드가 없다.

**해법**: `guides/08_병렬세션.md` 신규 작성. 내용:
- Git Worktree 기본(`git worktree add`)
- 본사 vs 사업부별 동시 claude 실행 패턴
- 언제 병렬(프론트/백, 리서치/초안), 언제 직렬(의존성 있는 경우)
- 2세션부터 시작 → 3~5개로 확장 로드맵

**기대효과**: 실전 생산성 2~3배. Boris 팀 #1 팁을 한국어 경영 맥락에 이식.

## 6. 반대 의견 (MUST Disagree 체크)

> "PDF 내용이 좋으니 전부 흡수하자" — **반대합니다. (확신도 90%)**

- **이유 1 (Inversion)**: PDF는 **개인 생산자** 관점이다. 우리는 **조직 시뮬레이션**이다. 42팁을 전부 번역하면 우리의 고유 포지셔닝(C-Suite 페르소나, Tier 프로토콜)이 희석된다.
- **이유 2**: Part 5 업종 프로젝트(랜딩페이지/블로그/MVP)는 우리 `projects/_template` + `guides/07`과 **중복**이다. 차용 X.
- **이유 3 (lessons.md 클러스터 A)**: "좋아 보이면 다 넣기" 패턴으로 과거 레포 비대화 사례 있음.

**따라서 차용 범위는**: ① 인프라 레이어(Hook/학습루프/병렬세션/비용설정) ② 방법론(Boris 42팁 Top 15) 까지. **업종 프로젝트, 도구 설치 튜토리얼, 수익화 상세는 차용 금지 또는 링크로만.**
