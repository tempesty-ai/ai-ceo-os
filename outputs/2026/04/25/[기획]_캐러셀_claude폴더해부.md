### 🤍 SLIDE 5 — Hooks (라이트 배경)

**상단 라벨 (테라코타)**
행동을 강제로 고정하는 방법 ②

**헤드라인**
CLAUDE.md는 제안,
Hooks는 강제입니다

**본문**
CLAUDE.md에 "린터 돌려줘" 써봤자
**가끔** 따릅니다.

Hook은 **매번** 실행됩니다.

주요 이벤트:
| Hook | 언제 |
|---|---|
| `PreToolUse` | 도구 실행 전 (보안 게이트) |
| `PostToolUse` | 도구 실행 후 (자동 포맷) |
| `Stop` | 작업 완료 전 (테스트 통과 확인) |

**포인트 박스 (테라코타 보더)**
⚠️ exit code 2 = 실행 차단
exit code 1 = 로그만 남김 (차단 안 됨!)
이게 가장 흔한 실수입니다.

### 🤍 SLIDE 7 — settings.json (라이트 배경)

**상단 라벨 (테라코타)**
허용/차단 명단 ④

**헤드라인**
Claude가 뭘 할 수 있고
없는지 정하는 곳

**본문**
```json
{
  "permissions": {
    "allow": [
      "Bash(npm run *)",
      "Bash(git status)",
      "Read", "Write", "Edit"
    ],
    "deny": [
      "Bash(rm -rf *)",
      "Bash(curl *)",
      "Read(./.env)"
    ]
  }
}
```

**3가지 상태:**
✅ allow → 묻지 않고 실행
❌ deny → 완전 차단
⬜ 그 외 → Claude가 먼저 물어봄
