# projects/ — AI 사업부 폴더

> 본사(ai-ceo-os 루트)가 회사 전체 헌법이라면, `projects/` 아래 각 폴더는 **사업부(부서)** 입니다.
> 이 문서는 "왜 쪼개는가 / 어떻게 쪼개는가 / 뭘 넣는가"를 5분 안에 이해시키는 것이 목표입니다.

---

## 1. 왜 쪼개나 — 본사 하나의 한계

본사 `CLAUDE.md`에 모든 규칙(마케팅/개발/운영/고객)을 다 넣으면:

- 파일이 비대해져 AI가 중요한 걸 놓칩니다.
- "인스타 릴스 기획해줘" 같은 요청에 범용 답변이 나옵니다.
- 부서별 금지행동/톤/KPI가 섞여서 충돌합니다.

**해결**: 본사는 회사 전체 헌법만 두고, **세부 맥락은 사업부 폴더로 내려보냅니다.**
사업부에서 작업하면 그 폴더의 `CLAUDE.md` + 담당 에이전트 + 스킬 + 룰이 **자동으로** 함께 로드됩니다.

### 본사 vs 사업부 역할 분담

| 구분 | 본사 (ai-ceo-os 루트) | 사업부 (projects/xxx/) |
|:---|:---|:---|
| 역할 | 전사 헌법 + 조직도 + 공용 자산 | 특정 업무 도메인의 실전 작업장 |
| CLAUDE.md | 회사 미션/비전/금지행동/톤 | 이 사업부만의 목표/담당자/작업 규칙 |
| 에이전트 | C-Suite 9명 (전사 임원) | 이 사업부 전담 1~3명 선택 |
| 스킬 | 공통 스킬 | 도메인 특화 스킬 |
| 비유 | 사장님 집무실 | 마케팅팀 사무실 / 고객센터 / 개발실 |

---

## 2. 하네스(Harness) 5종 — 사업부 폴더 안에 뭘 넣나

> 말을 탈 때 고삐·안장·등자가 필요한 것처럼, AI에게 일을 시키려면 "장치 세트"가 필요합니다.
> 이 5개가 꽉 물려 있어야 AI가 그 사업부에서 "자기 일"처럼 일합니다.

```
projects/marketing/
├── CLAUDE.md      ← ① 사업부 헌법 (이 팀의 미션/룰)
├── agents/        ← ② 전담 직원 (C-Suite에서 1~3명 선택)
├── skills/        ← ③ 업무 매뉴얼 (이 팀이 쓰는 SOP)
├── context/       ← ④ 참고 자료 (채널 규칙, 고객 여정 등)
└── rules/         ← ⑤ 자동 적용 규칙 (금지/품질/의사결정)
```

| # | 하네스 | 역할 | 없으면 |
|:---:|:---|:---|:---|
| ① | `CLAUDE.md` | 이 사업부의 미션/KPI/판단 기준 | 팀이 방향 없이 떠돔 |
| ② | `agents/` | 이 사업부 담당 임원 명단 + 역할 | "누가 할 일인데요?" |
| ③ | `skills/` | 반복 작업의 SOP | 매번 처음부터 설명 |
| ④ | `context/` | 이 팀만의 세부 정보(채널 규칙 등) | 결과가 범용화 |
| ⑤ | `rules/` | 자동 적용 금지/품질 룰 | 사고 반복 |

## 3. 쪼개기 3대 원칙

### 원칙 1. 업무 카테고리로 쪼갠다 (채널 X)

| 비교 | ❌ 채널로 쪼개기 | ✅ 업무로 쪼개기 |
|:---|:---|:---|
| 구조 | insta/ · youtube/ · blog/ | marketing/ · dev/ · operations/ · customer/ |
| 문제 | "릴스 기획" 스킬을 3개 폴더에 복붙 | 재사용 가능, 협업 경계 명확 |

채널별 데이터는 `context/channel-rules.md`에서 구분하면 됩니다.

### 원칙 2. 처음엔 2~3개로 시작한다

한 번에 6개 만들면 다 무너집니다. **가장 매출 직결되는 1개**부터 완성하고,
돌려보면서 체득한 다음 나머지를 추가하세요.

### 원칙 3. 하나 잘 만들고 복제한다

첫 사업부는 30~40분, 두 번째는 10분, 세 번째는 5분. **4번째부턴 사실상 무료**입니다.
그래서 첫 하나(=`_template/`)를 **대충 만들면 안 됩니다**.

## 4. 시작하는 법 — 2가지 경로

### 경로 A. 템플릿 복제 (권장)

```bash
# _template 을 복제해서 내 첫 사업부 만들기
cp -r projects/_template projects/marketing

# 그 다음 Claude 에게:
#   "projects/marketing/CLAUDE.md 의 {{슬롯}} 을 내 사업(네일샵)에 맞춰 채워줘"
```

### 경로 B. AI에게 맡기기

```
"projects/marketing 폴더를 만들어줘.
 _template 구조를 복제하고, 내 사업(네일샵 예약 유입)에 맞춰
 CLAUDE.md / agents / skills / context / rules 를 채워줘."
```
