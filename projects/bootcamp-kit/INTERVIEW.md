# INTERVIEW.md — AI가 나를 인터뷰한 기록

> **사용법**: `prompts/01-interview-setup.md` 복사 → Claude Code에 붙여넣기 → 인터뷰 시작
> AI가 **10개 질문을 한 번에 1개씩** 던지고, 당신은 **답변만** 하면 됩니다.
> 10분 안에 기획이 완성됩니다.

---

## 🎯 왜 AI에게 인터뷰를 시키는가

- 혼자 PLAN.md를 채우려면 **백지 공포증** 옴
- AI에게 "날 인터뷰해달라"고 하면 → **답만 하면 됨** → 훨씬 쉬움
- AI가 내 답변을 듣고 **적절한 다음 질문**을 생성 → 기획이 입체화

**대표님 10습관 #8**: "선택지 2개 비교" — 인터뷰 질문에도 적용

---

## 📝 인터뷰 기록

### Q1: 누구를 위한 서비스인가?
**A1:** 품질 관리자, QA 담당자, 애플리케이션 테스트가 필요한 사람

### Q2: 그 사람들이 지금 겪는 가장 큰 문제는?
**A2:** 테스트 계획서 만들기, Test Case 만들기

### Q3: 지금은 어떻게 해결하고 있나?
**A3:** 팀 내 구글 드라이브를 통해서 수동으로 입력해서 만들고 있음

### Q4: 그 현재 해결책의 단점은?
**A4:** 하나하나 다 입력해야 해서 시간이 많이 걸린다. 인간이 하는 거라 실수가 나올 수 있다.

### Q5: 이 서비스만의 차별점은?
**A5:** 테스트가 필요한 애플리케이션이나 정보를 넣을 경우 자동으로 Test Case를 뽑아준다.

### Q6: 핵심 기능 3개만 고르면?
**A6:** A, B, C — 앱/기능 설명 입력 → Test Case 자동 생성 / 테스트 계획서 자동 생성 / 편집 + 구글 드라이브 내보내기

### Q7: 무료인가 유료인가? 가격은?
**A7:** 무료는 기본 Test Case 제공, 유료는 상세한 Test Case + 테스트 계획서

### Q8: 수익화 경로는?
**A8:** 무료로 써서 퀄리티가 괜찮으면 유료 결제를 통해 수익화

### Q9: 3개월 후 이 서비스의 모습은?
**A9:** 증가하는 매출을 보면 많이 사용되고 있구나라고 생각할 수 있을 것 같음

### Q10: 실패를 어떻게 정의할까?
**A10:** 매출이 없는 경우에 그런 생각을 해볼 것 같음

## 🎁 AI 요약 (인터뷰 끝난 후 AI에게 요청)

```
위 10개 답변을 기반으로 내 PLAN.md를 채워줘.
특히:
- 한 줄 정의 (1번)
- 타겟 (2번)
- 핵심 기능 3개 (6번)
- 수익화 경로 (8번)
- 생산자 3원칙 자가 답변
```

→ 결과물을 `PLAN.md`에 붙여넣기

## 💡 답변 팁

- "잘 모르겠어요" OK. AI가 다른 각도로 질문해줌.
- 답변이 너무 짧으면 AI가 "좀 더 구체적으로" 되물음.
- **솔직하게**. 있어보이는 답 금지 — AI가 진짜 기획 못 만들어줌.
