# PLAN.md — 내 프로젝트 기획서

> **대표님 10습관 #1**: "기획서 없이 코드 안 씀"
> 이 파일 없이 Claude Code 켜지 말 것.

---

## 1. 한 줄 정의

"QA 담당자가 앱/기능 정보를 입력하면 AI가 테스트 계획서와 Test Case를 자동으로 생성해주는 웹 서비스"

---

## 2. 타겟 (누구?)

- **핵심 대상**: QA 담당자, 품질 관리자, 애플리케이션 테스트가 필요한 개발팀
- **이들이 지금 겪고 있는 문제**:
  - 문제 1: 테스트 계획서와 Test Case를 하나하나 수동으로 입력 → 시간 과다 소요
  - 문제 2: 사람이 직접 작성하다 보니 누락·실수 발생
  - 문제 3: 팀 내 구글 드라이브로 공유하지만 협업 효율 낮음
- **이 문제를 지금 어떻게 해결?** 구글 드라이브에서 수동 입력 — 시간이 많이 걸리고 휴먼 에러 발생

---

## 3. 해결 (무엇을?)

- **이 서비스가 해결하는 문제 1문장**: 테스트 대상 정보를 입력하면 AI가 테스트 계획서와 Test Case를 자동으로 생성해 시간과 실수를 동시에 줄인다
- **Before** (서비스 없을 때): 구글 드라이브 열고 → 항목 하나씩 직접 입력 → 검토 → 수정 (수 시간 소요)
- **After** (서비스 쓴 후): 앱/기능 설명 입력 → AI가 계획서+Test Case 자동 생성 → 편집 후 내보내기 (수십 분으로 단축)

---

## 4. 핵심 기능 (최대 3개)

> 3개 이상 쓰면 망한다. MVP는 3개만.

1. **기능 1**: 앱/기능 설명 입력 → Test Case 자동 생성 (AI)
2. **기능 2**: 테스트 계획서 자동 생성 (AI)
3. **기능 3**: 생성된 Test Case 편집 + 구글 드라이브 내보내기

**V1에서 뺀 기능 (v2 이후)**:
- 테스트 결과 기록 + 통계
- 기존 Test Case 재사용/템플릿화
- 팀 협업 기능

---

## 5. 차별점

- **이미 비슷한 서비스**: TestRail, Zephyr, Qase.io
- **내 차별점**: AI가 테스트 대상 설명만 넣으면 계획서+Test Case를 한 번에 생성 — 기존 툴은 직접 입력 방식

---

## 7. 수익화 경로

- **상품명**: Free / Pro 플랜
- **가격**:
  - Free: 기본 Test Case 생성 (제한된 개수/상세도)
  - Pro: 상세 Test Case + 테스트 계획서 전체 기능 (월정액 예정)
- **누가 살까?** 무료로 써보고 퀄리티에 만족한 QA 담당자/팀
- **첫 매출 시나리오**: 무료 플랜 사용 → 한도 초과 or 더 상세한 결과 필요 → Pro 결제
- **Kill 기준**: 3개월 후 유료 전환 0명이면 피벗 또는 종료 검토

## 8. 생산자 3원칙 자가 답변

> 4-2 과제 C-08 필수 제출. 5-2 과제 C-10 최종 재점검.

1. **문제 먼저** — "누가 돈 내고 쓸 문제인가?"
   → QA 담당자가 Test Case 작성에 쓰는 시간(수 시간)을 수십 분으로 줄이는 문제. 기업에서 QA 인건비 절감 직결.

2. **가치 먼저** — "다시 올 이유가 있는가?"
   → 매 스프린트마다 새 기능이 나올 때마다 Test Case가 필요 → 반복 사용 구조

3. **수익 먼저** — "좋아요 100보다 결제 1건"
   → 무료 한도 초과 시 자연스럽게 유료 전환 유도. 팀 단위 구독으로 확장 가능.
