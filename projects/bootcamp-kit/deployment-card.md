## ⚠️ 배포 실수 TOP 3 (대표님이 실제로 터뜨린 것)

### ❌ 실수 1: API 키 코드에 박아놓기
```ts
// 🚫 절대 금지
const API_KEY = "sk-ant-abc123...";
```
```ts
// ✅ 올바른 방법
const API_KEY = process.env.CLAUDE_API_KEY;
```

### ❌ 실수 2: `git add -A`로 `.env.local`까지 커밋
- **대표님 실제 사고**: 2026-04-11 `.env.backup_20260410`이 GitHub에 3주간 커밋됨
- **방어**: `.gitignore`에 `.env*` 명시 + `git status`로 확인 후 `git add` 파일 지정

### ❌ 실수 3: HTTP 200 = 배포 완료로 착각
- **대표님 실제 사고**: 2026-04-14 Observatory HTTP 200인데 실제 콘텐츠는 SPA fallback 668바이트
- **방어**:
  1. HTTP 상태 코드 (200)
  2. `Content-Length` (실제 파일 크기와 일치)
  3. 본문 첫 200자에 **고유 문자열** 존재 여부

## 🎯 배포 전 필수 체크리스트

`prompts/04-safety-checklist.md` 참조.

배포 버튼 누르기 전에 AI에게 6가지 안전 체크 시키기:
- [ ] `.env.local`이 `.gitignore`에 있는가?
- [ ] API 키가 코드에 하드코딩되어 있지 않은가?
- [ ] `git status`로 의도치 않은 파일이 스테이징되어 있지 않은가?
- [ ] localhost:3000에서 실제로 작동하는가?
- [ ] `npm run build` 에러 없이 빌드되는가?
- [ ] 404/500 에러 페이지가 있는가?

**6개 다 통과 → 배포 OK**
**1개 실패 → 수정 후 재체크**
**3개 실패 → Plan Mode 재검토**
