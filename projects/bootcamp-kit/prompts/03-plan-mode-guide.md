# 03 — Plan Mode 활용 가이드

> **Plan Mode**: Claude Code가 **코드 쓰기 전 계획부터** 보여주는 기능.
> 대표님 10습관 #1 "기획서 없으면 코드 안 씀"의 **작업 단위 버전**.

---

## 🎯 언제 쓰나

- 새 기능 추가할 때 (작은 거든 큰 거든)
- 기존 코드 큰 수정할 때
- "이거 어떻게 만들지?" 막힐 때
- **리팩토링 / 버그 수정 / 성능 개선 등 3단계 이상 작업**

---

## 🚀 사용법 (Claude Code)

### 방법 1: 키보드 단축키
```
Shift + Tab
```
(한 번 누르면 Plan Mode 진입. 상단에 `[PLAN MODE]` 표시)

### 방법 2: 프롬프트
```
Plan Mode로 진입해. 내 작업 계획을 먼저 세워줘.
```

### Plan Mode에서 나오기
```
이 계획으로 진행해줘.
```
또는 `Ctrl + Enter`

---

## 🆚 Before / After 비교

### ❌ Plan Mode 없이
```
You: "블로그 만들어줘"

AI: (바로 코드 생성)
    → 제네릭하고 내 맘에 안 드는 결과물
    → 뒤늦게 "아 이거 말고..."
    → 3번 재작성 → 시간 3배
```

### ✅ Plan Mode 있이
```
You: "블로그 만들어줘"

AI: [PLAN MODE] 10단계 계획을 세웠습니다:
    1. Next.js 프로젝트 초기화
    2. MDX 파싱 설정
    3. 포스트 디렉토리 구조
    4. 홈페이지 (포스트 목록)
    5. 상세 페이지 라우팅
    6. 마크다운 렌더링
    7. 목차 자동 생성
    8. 다크모드 토글
    9. SEO 메타데이터
    10. 배포 설정

    이 계획대로 진행해도 될까요?

You: "3번은 `/posts/*.mdx`로 바꿔줘. 7번 빼. 나머지 OK."

AI: (수정된 계획대로 **정확하게** 구현)
```

→ **결과물 품질 3배 상승. 재작성 0.**

## 📊 Plan Mode vs 기획서 (PLAN.md)

| 레벨 | 파일/기능 | 범위 |
|:---:|:---|:---|
| **프로젝트** | `PLAN.md` | 전체 서비스 기획 (한 번만) |
| **작업** | Plan Mode | 각 기능/버그 단위 (매번) |

**두 개 다 있어야 이중 안전.**

---

## 🔗 대표님 10습관 연결

| 습관 | Plan Mode에서 |
|:---|:---|
| #1 기획서 없으면 코드 안 씀 | Plan Mode = 작업별 기획서 |
| #2 "이게 진짜 문제인가?" | 계획 단계에서 목표 재정의 |
| #3 "다른 사람은 어떻게 했지?" | AI에게 "비슷한 프로젝트 찾아봐" 먼저 시킴 |
| #8 선택지 2개 비교 | Plan Mode에서 "A안 B안 비교" 요청 |

---

## ⚠️ Plan Mode 안 쓰면 안 되는 경우

- **DB 스키마 바꿀 때** (역산 불가능)
- **결제 로직 추가할 때** (대표님 실전 사고 직결)
- **프로덕션 배포 전**
- **API 호출 비용 드는 작업** (Claude API, 토스페이먼츠)

→ 이럴 때 Plan Mode 없으면 **돈+시간이 이중으로 날아감**.
