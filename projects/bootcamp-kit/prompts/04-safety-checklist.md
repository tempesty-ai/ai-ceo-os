# 04 — 배포 전 안전 체크리스트 프롬프트

> 배포 버튼 누르기 전 AI에게 **6가지 안전 체크**를 시키는 프롬프트.
> 대표님 10습관 #7 "테스트 안 했으면 완료 아님" 자동화 버전.

---

## 🎯 복사해서 쓸 프롬프트

```
배포 전 안전 체크를 해줘. 아래 6개를 순서대로 확인하고 결과를 표로 알려줘.

1. .env.local 파일이 .gitignore에 있는가?
2. API 키가 코드에 하드코딩되어 있지 않은가? (grep으로 확인)
3. git status로 의도치 않은 파일이 스테이징되어 있지 않은가?
4. localhost:3000에서 실제로 작동하는가? (curl로 확인)
5. npm run build 에러 없이 빌드되는가?
6. 404, 500 에러 페이지가 있는가?

각 항목을 실제로 확인하고, 아래 표로 답해줘:

| # | 체크 | 통과/실패 | 근거 |
|:--|:---|:---:|:---|

실패가 있으면 수정 코드도 제안해줘.
```

## 📋 결과 해석

### ✅ 6개 모두 통과
→ **Vercel Deploy 버튼 OK**.
```bash
git push
```

### ⚠️ 1~2개 실패
→ AI가 제안한 수정 먼저 적용 → 체크리스트 **재실행**

### 🚨 3개 이상 실패
→ **Plan Mode 진입** → 전체 재검토.
```
배포 전 안전 체크에서 3개 이상 실패. Plan Mode로 재설계 필요.
```

## ⏱ 타임박스

- 안전 체크: **3분**
- 실패 항목 수정: **항목당 5~10분**
- 재체크: **3분**
- **배포 전 총 10~20분**

**Deploy 버튼 누르는 데 10분 걸리는 게 정상**. "3초만에 배포"는 사고의 지름길.

---

## 🎯 배포 일상 루틴

```
매일 배포 루틴 (5-2 이후):

1. git add [특정 파일들]       ← git add -A 금지
2. git status                  ← 스테이징 확인
3. 위 프롬프트로 AI 안전 체크  ← 이 파일
4. 6/6 통과 시 git commit
5. git push                    ← 자동 배포
6. Vercel 배포 화면 열어 URL 열어봄  ← 10습관 #7
7. 고유 문자열 확인            ← HTTP 200 ≠ 배포 완료
```

**이 7단계 루틴이 내 습관이 되면 대표님 수준.**
