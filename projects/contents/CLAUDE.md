# 콘텐츠 사업부 CLAUDE.md

> 이 사업부의 헌법. 본사 `CLAUDE.md`의 채널 전략과 연결되되,
> **조회수·바이럴·인기** 지표를 최우선으로 다룬다.

---

## 미션

조회수 잘 나오는 콘텐츠, 인기 있는 콘텐츠를 지속적으로 생산하는 **콘텐츠 제작 시스템**을 운영한다.
"한 번 터지는 게 아니라, 매주 터지는 구조를 만든다."

---

## 이 사업부가 책임지는 일

1. **바이럴 콘텐츠 기획** — 트렌드 리서치 + 후킹 설계 + 포맷 선택
2. **멀티플랫폼 콘텐츠 생산** — 인스타 릴스 / 유튜브 숏츠·롱폼 / 틱톡
3. **콘텐츠 성과 예측** — 업로드 전 바이럴 가능성 사전 검증
4. **주간 콘텐츠 캘린더** — 플랫폼별 발행 계획 + 주제 배분
5. **성과 패턴 분석** — 터진 콘텐츠 해부 → 재현 공식화

---

## 담당 임원

- **주 담당**: CCO — 콘텐츠 기획·후킹·포맷 총괄
- **부 담당**: CDO — 바이럴 패턴 데이터 분석·성과 예측
- **+1**: CMO — 도달률·알고리즘 최적화·바이럴 확산

상세는 `agents/README.md` 참조.

---

## 성공 지표 (KPI)

- **조회수**: 릴스/숏츠 평균 조회수 (목표치는 데이터 쌓이면 설정)
- **시청 완료율**: 알고리즘 핵심 지표
- **저장·공유율**: 바이럴 확산 지표
- **도달 대비 팔로우 전환율**: 조회수가 팔로워로 이어지는 비율
- **터진 콘텐츠 비율**: 전체 발행 중 평균 2배 이상 조회수 달성 비율

## 크로스 사업부 협업 프로토콜

**참조**: `projects/통합-오케스트레이터.md`

### 언제 marketing 사업부와 협업하는가
- 콘텐츠 생산 시: contents가 기획한 콘텐츠를 **marketing이 팔로우 전환 관점 검수**
- 주간 캘린더 시: marketing의 고객 여정 배분 + contents의 바이럴 최적화 교차
- 성과 분석 시: contents CDO 조회수 + marketing CDO 전환율 교차 분석

### contents가 marketing에 요청하는 것
- "이 대본이 팔로우 전환으로 이어지는가?" → marketing CCO
- "이 콘텐츠 고객 여정 몇 단계용인가?" → marketing CMO

### marketing이 contents에 요청하는 것
- "이 주제 조회수 관점에서 후킹 강화해줘" → contents CCO
- "이 콘텐츠 바이럴 가능성 예측해줘" → contents CDO

### 단독 실행 조건
조회수·바이럴·공유율만 관련된 작업 → contents 단독 실행 (통합 오케스트레이터 불필요)
