# 콘텐츠 사업부 품질 기준선

> "조회수 잘 나오는 콘텐츠"의 최소 조건.
> 이 기준을 통과하지 못한 콘텐츠는 납품하지 않는다.

---

## 릴스·숏츠·틱톡 (숏폼 공통)

- [ ] **첫 3초 후킹**: 질문/충격/반전/결과 먼저 중 하나로 시작 (인사·서론 없음)
- [ ] **길이**: 30~60초 (플랫폼 최적값 내)
- [ ] **후킹 3개 이상**: 서로 다른 타입의 후킹이 영상 내 3개 이상 배치
- [ ] **CTA 1개만**: 저장 / 팔로우 / 댓글 중 정확히 1개
- [ ] **전문용어 풀어쓰기**: AI 용어 사용 시 "쉽게 말하면..." 처리
- [ ] **구체적 사례/숫자**: 추상적 주장 대신 구체적 예시 1개 이상
- [ ] **3버전 생산**: 서로 다른 후킹 전략의 버전 3개 (복붙 변형 금지)
- [ ] **CEO 톤**: `context/ceo-persona.md` 규칙 준수

---

## 유튜브 롱폼

- [ ] **썸네일·제목**: 클릭하고 싶은 제목 + CTR 최적화 썸네일
- [ ] **인트로 60초 이내**: 60초 안에 "이 영상이 왜 가치 있는지" 전달
- [ ] **챕터 구분**: 10분 이상 영상은 챕터 마커 필수
- [ ] **CTA 배치**: 초반(구독 요청) + 중반(저장/알림) + 마지막(다음 영상 연결)
- [ ] **SEO 최적화**: 제목·설명란·태그에 핵심 키워드 포함

---

## 콘텐츠 기획서 (캘린더·배치 기획)

- [ ] **콘텐츠 믹스 비율 명시**: 바이럴 50% / 저장 30% / 트렌드 20%
- [ ] **플랫폼 명시**: 각 콘텐츠의 타겟 플랫폼 지정
- [ ] **후킹 전략 명시**: 어떤 후킹 타입 사용할지 미리 설계
- [ ] **발행 타이밍**: 플랫폼별 최적 업로드 시간 지정 (`context/platform-rules.md` 기준)

---

## 바이럴 성과 예측 보고서

- [ ] **데이터 출처 명시**: Zero-Inference 원칙 — 근거 없는 수치 0건
- [ ] **3가지 신호 교차 검증**: 후킹 강도 + 트렌드 일치도 + 구조 완성도
- [ ] **불확실 가설 레이블**: "검증 필요" 명시
- [ ] **개선 포인트**: 있으면 1~3개 구체적 제안

---

## 공통 품질 기준

- [ ] **AI 정보 검증**: 확인되지 않은 AI 기능·스펙 단정 0건
- [ ] **낚시성 후킹 없음**: 후킹이 콘텐츠 본문과 일치
- [ ] **타겟 페르소나 명시**: "누구를 위한 콘텐츠인지" 명확
- [ ] **금지행동 0건**: `rules/금지행동.md` 10개 항목 자동 체크 완료
