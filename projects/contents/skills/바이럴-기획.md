# 바이럴 콘텐츠 기획 스킬

> 조회수 잘 나오는 콘텐츠를 체계적으로 기획하는 파이프라인.
> 후킹 공식 + 바이럴 메커니즘 + CEO 톤앤매너를 3버전으로 생산.

---

## 0단계: 자산 로드

### 필수 로드
- `CLAUDE.md` (사업 맥락, CEO 톤앤매너 3원칙)
- `projects/contents/CLAUDE.md` (콘텐츠 사업부 헌법 — 콘텐츠 믹스, KPI)
- `projects/contents/context/viral-formula.md` (바이럴 공식 라이브러리)
- `projects/contents/context/platform-rules.md` (플랫폼별 운영 규칙)
- `projects/contents/rules/금지행동.md` (금지 목록 자동 체크)
- `context/ceo-persona.md` (CEO 톤앤매너 상세)

### 조건부 로드
- `tasks/lessons.md` — 과거 콘텐츠 교훈
- `tasks/todo.md` — 이번 주 콘텐츠 계획과 연결

---

## 1단계: 주제 + 타겟 명확화

사용자 입력: `$ARGUMENTS`

아래 4가지가 입력에 없으면 **1번만** 역질문:

1. **타겟**: AI+인사이트 / AI+육아 중 어느 주제인가?
2. **콘텐츠 유형**: 바이럴 후킹형 / 저장·신뢰 자산형 / 트렌드 편승형?
3. **플랫폼**: 인스타 릴스 / 유튜브 숏츠 / 틱톡 / 멀티?
4. **길이**: 30초 / 45초 / 60초?

정보가 충분하면 역질문 생략하고 즉시 기획 시작.

**기본값** (정보 없을 때):
- 타겟: AI+인사이트
- 유형: 바이럴 후킹형
- 플랫폼: 인스타 릴스
- 길이: 45초

---

## 2단계: 바이럴 메커니즘 선택

`context/viral-formula.md`의 5가지 바이럴 메커니즘에서 **최소 2개** 선택:

1. 사회적 화폐 (잘 알려지지 않은 정보)
2. 공감 고통 (공통 불편함의 언어화)
3. 실용 가치 (즉시 써먹는 꿀팁)
4. 스토리 텐션 (반전·의외의 결과)
5. 집단 정체성 (특정 집단의 경험 공유)

**선택 근거 명시**: "이 콘텐츠가 왜 퍼지는가 1줄"

---

## 3단계: 역순 대본 설계 (Backward Script)

각 버전마다 아래 순서로 설계:

```
[A] 최종 감정/행동 1줄 정의 (시청자가 끝나고 무엇을 하는가)
  ↓
[B] 핵심 메시지 1줄
  ↓
[C] 전환 스토리 (2~3문장)
  ↓
[D] 후킹 3개 이상 설계 (첫 3초에 집중)
  ↓
[E] CTA 1개 (정확히 1개)
```

---

## 4단계: 후킹 공식 적용 (최소 3개)

`context/viral-formula.md`의 후킹 12타입에서 **3개 이상 조합**:

질문형 / 충격 통계 / 반전 / 공감 고통 / 결과 먼저 / 비밀 공개 / 부정 어법 / 권위 인용 / 시간 제한 / 리스트 / 자기 반성 / 비교

**후킹 금지 패턴** (즉시 재작성):
- "안녕하세요 여러분~"
- "오늘은 ___에 대해 알아볼게요"
- "AI가 세상을 바꿀..."
- 3초 안에 "왜 봐야 하지?" 답 안 나오는 인트로

---

## 5단계: CEO 톤앤매너 적용

`context/ceo-persona.md` 규칙 100% 준수:
- 존댓말, 간결·명확한 전문가 톤
- 전문용어 후 반드시 "쉽게 말하면..." 풀어쓰기
- 거창한 서론 0건, 즉시 가치 전달

---

## 6단계: 3버전 출력

### 버전 1: [후킹 전략명]

```markdown
- 바이럴 메커니즘: [선택한 2개]
- 후킹 타입: [사용한 3개]
- 타겟: [구체적 페르소나]
- 콘텐츠 유형: [바이럴/저장/트렌드]
- 플랫폼: [인스타/유튜브/틱톡]
- 길이: [30/45/60초]

---
[대본 본문]
---
예상 반응: [시청자가 어떤 행동을 할 것인가]
```

버전 2, 3도 동일 형식. **서로 다른 후킹 타입·구조·톤 사용 (복붙 변형 금지).**

### 플랫폼별 캡션 3버전

```markdown
### 캡션 1 ([버전 1]용)
[1~3줄 본문]
[CTA 1개]
[해시태그: platform-rules.md 기준]
```

---

## 7단계: 품질 자체 검증

제출 전 체크:

- [ ] 후킹 3개 이상 명시?
- [ ] 첫 문장이 3초 내 후킹 (인사 없음)?
- [ ] CTA 정확히 1개?
- [ ] CEO 톤 일치?
- [ ] 3버전 서로 다른 후킹/구조?
- [ ] 금지행동 0건?
- [ ] 미검증 AI 정보 0건?

미통과 → 재작성. 체크 우회 금지.
