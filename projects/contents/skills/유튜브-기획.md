# 유튜브 콘텐츠 기획 스킬

> 조회수 잘 나오는 유튜브 영상을 체계적으로 기획하는 파이프라인.
> 숏츠(60초 이하)와 롱폼(5분 이상) 두 포맷을 모두 커버.

## 1단계: 포맷·주제 명확화

사용자 입력: `$ARGUMENTS`

아래 정보가 없으면 **1번만** 역질문:

1. **포맷**: 숏츠(60초 이하) / 롱폼(5분 이상) 중 어느 것?
2. **주제**: AI+인사이트 / AI+육아?
3. **목적**: 신규 시청자 발견 / 기존 구독자 심화 콘텐츠?

기본값: 숏츠 / AI+인사이트 / 신규 발견

---

## 2단계: SEO 키워드 선정

유튜브는 **검색 기반 도달**이 강하다.

키워드 선정 기준:
1. **검색 의도 파악**: 시청자가 무엇을 해결하고 싶어서 검색하는가?
2. **경쟁도 파악**: 대형 채널이 이미 도배한 키워드는 피하거나 앵글 특화
3. **롱테일 키워드**: "AI 활용법"보다 "초보자도 되는 AI 활용법 3가지" 형태

출력: 메인 키워드 1개 + 보조 키워드 2~3개

---

## 3단계: 썸네일·제목 설계 (숏츠/롱폼 공통)

### 제목 공식
- **숫자 포함**: "3가지", "5분 안에", "10배 빠른"
- **혜택 명시**: 무엇을 얻는가
- **호기심 유발**: 반전 또는 "몰랐던" 앵글
- **키워드 자연 삽입**: 첫 부분에 배치

**제목 3개 생산** (서로 다른 앵글):
- 버전 A: 결과 먼저형 ("이것만 알면 AI 10배 잘 씀")
- 버전 B: 호기심형 ("아무도 안 알려준 AI 꿀팁")
- 버전 C: 문제 해결형 ("AI가 틀린 답 낼 때 해결법")

### 썸네일 컨셉 기획
```
메인 비주얼: [한 줄]
텍스트 오버레이: [강조 문구, 3~5단어]
색상·톤: [밝음/어두움, 주요 색상]
표정/포즈: [있다면]
```

---

## 4단계: 스크립트 구조 기획

### 숏츠 구조 (60초 이하)
```
[0~3초] 후킹 (질문/충격/반전 중 하나)
[3~50초] 핵심 내용 (1가지만 — 여러 개 금지)
[50~60초] CTA (구독 or 저장 중 1개)
```

### 롱폼 구조 (5분 이상)
```
[0~60초] 인트로 후킹 + "이 영상에서 얻는 것 3가지"
[1~N분] 챕터 구분 (한 챕터 = 한 가지 내용)
  - 챕터마다: 핵심 포인트 → 실전 예시 → 미니 정리
[마지막 30초] 핵심 요약 + CTA (구독 + 다음 영상 예고)
```

**롱폼 챕터 설계**:
| 챕터 | 내용 | 예상 시간 |
|:---|:---|:---:|
| 1 | [핵심 포인트 1] | [분] |
| 2 | [핵심 포인트 2] | [분] |
| 3 | [핵심 포인트 3] | [분] |

---

## 5단계: 설명란·해시태그

### 설명란 구조 (SEO 최적화)
```
[1~3줄] 키워드 포함 요약 (가장 중요 — 검색에 반영)
[4~10줄] 주요 타임스탬프 (롱폼의 경우)
[하단] 채널 소개 + 소셜 링크
```

### 해시태그
- 숏츠: 3~5개 (플랫폼 규칙에서 가져옴)
- 롱폼: 5~8개
