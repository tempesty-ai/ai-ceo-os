# 릴스 대본 작성 스킬

> CEO의 톤앤매너에 100% 맞는 릴스/숏폼 대본을 **3버전 + 캡션 3버전** 동시 생성.
> 본사(에이나우) 내부 운영 스킬을 수강생용으로 포팅한 버전입니다.

---

## 0단계: 어떤 파일을 읽는가

### 필수 로드
- `CLAUDE.md` (사업 맥락, 고객 여정 6단계)
- `projects/marketing/CLAUDE.md` (마케팅 사업부 헌법 — KPI, 담당, 콘텐츠 비율)
- `context/ceo-persona.md` (CEO 톤앤매너 — 말투, 금지 표현, 말끝 스타일)
- `context/channel-rules.md` (채널별 규칙 — 해시태그, CTA 방식)
- `projects/marketing/rules/금지행동.md` (사업부 금지 목록 — 9개 항목 자동 체크)

### 조건부 로드
- `knowledge/channel-data/` 하위 본인의 실제 릴스 인사이트(있으면) — 검증된 후킹 패턴 추출
- `tasks/lessons.md` — 콘텐츠 관련 과거 교훈
- `tasks/todo.md` — 이번 주 콘텐츠 계획과의 연결

**없으면?** 없어도 작동하지만 품질이 떨어집니다. `context/ceo-persona.md`는 최소 요구사항.

---

## 1단계: 주제 + 타겟 명확화

사용자가 던진 주제: `$ARGUMENTS`

아래 4가지가 입력에 명시되어 있지 않으면 **1번만** 역질문:

1. **타겟**: 누가 이 릴스를 봐야 하나? (왕초보 / 직장인 / 대표 / 프리랜서 중 1개 선택)
2. **목표**: 도달 확장 / 리드 수집 / 구매 전환 중 무엇?
3. **고객 여정 단계**: CLAUDE.md의 고객 여정 6단계 중 어디? (현재 1~3단계 집중)
4. **길이**: 30초 / 45초 / 60초?

정보가 충분하면 역질문 생략하고 즉시 작성.

> **벤치마크 참고**: `projects/marketing/CLAUDE.md`의 벤치마크 계정 목록에서 유사 콘텐츠 패턴을 확인하고 반영.

---

## 2단계: 역순 대본 작성법 (Backward Script)

**원칙**: 먼저 "이 릴스가 끝났을 때 시청자가 느끼는 감정/행동" 1줄을 정한 후, 거슬러 올라가 후킹을 설계한다.

```
[Step A] 최종 감정/행동 1줄 정의
  ↓
[Step B] 핵심 메시지 1줄
  ↓
[Step C] 전환 스토리 (2~3문장)
  ↓
[Step D] 후킹 3개 이상 설계 (첫 3초에 발사)
  ↓
[Step E] CTA 1개 (정확히 1개)
```

---

## 3단계: 후킹 공식 (최소 3개 적용)

### 후킹 12타입 (이 중 3개 이상 조합)

| # | 타입 | 예시 패턴 |
|:---:|:---|:---|
| 1 | **질문형** | "근데 왜 아직도 ___하고 있어요?" |
| 2 | **충격 통계** | "지금부터 ___하는 사람이 N%밖에 없대요" |
| 3 | **반전** | "다들 ___가 좋다는데, 사실 반대예요" |
| 4 | **공감 고통** | "___할 때마다 속 터지는 거 저만 그래요?" |
| 5 | **결과 먼저** | "이거 하나로 한 달에 ___ 벌었어요" |
| 6 | **비밀 공개** | "아무도 안 알려주는 ___의 진짜 이유" |
| 7 | **부정 어법** | "___만은 절대 하지 마세요" |
| 8 | **권위 인용** | "[유명인]이 ___라고 했다는데" |
| 9 | **시간 제한** | "딱 30초만 보세요. ___ 알려드릴게요" |
| 10 | **리스트** | "___하는 3가지 방법" (단, 뻔하면 감점) |
| 11 | **자기 반성** | "저도 예전엔 ___ 했거든요" |
| 12 | **비교** | "A 하는 사람 vs B 하는 사람" |

**금지**:
- "안녕하세요 여러분~" 같은 인사
- "오늘은 ___에 대해 알아볼게요" 같은 예고편
- "AI가 세상을 바꿀..." 류 거창한 서론
- 3초 안에 "왜 내가 이걸 봐야 하지?" 답이 안 나오는 인트로

---

## 4단계: CEO 톤앤매너 적용

`context/ceo-persona.md`에서 읽은 규칙을 100% 준수:

- **말투**: (예: "친근한 과외 선생님", "냉정한 실무자" — CEO 페르소나 파일에서 가져옴)
- **금지 표현**: (예: "여러분~", "~입니당", 특정 이모지 금지 등)
- **말끝 스타일**: (예: "~예요", "~입니다", "~거든요" 등 반복 스타일)
- **"쉽게 말하면" 규칙**: 전문 용어 쓸 때 반드시 **"쉽게 말하면..."** 풀어쓰기 붙이기

---

## 5단계: 출력 형식

### A. 대본 3버전 (서로 다른 후킹 전략)

```markdown
### 대본 1: [제목]
- 유형: [교육형/트렌드형/논쟁형/스토리형]
- 타겟: [구체적 타겟 1줄]
- 후킹: 
  1. (후킹 타입) - (문장)
  2. (후킹 타입) - (문장)
  3. (후킹 타입) - (문장)
- 길이: 30~60초
- 글자 수: 180~250자

---
[대본 본문]
---
예상 반응: [1줄]
```

대본 2, 3도 동일 형식. **서로 다른 후킹 타입/구조/톤**을 사용할 것. 복붙 변형 금지.

### B. 캡션 3버전 (각 대본에 매칭)

```markdown
### 캡션 1 (대본 1용)
[1~3줄 본문]
[CTA 1개]
해시태그: (채널 규칙에 따름. channel-rules.md 참조)
```

## 6단계: 품질 자체 검증 (필수)

제출 전 각 대본에 대해 체크:

- [ ] 후킹 3개 이상 명시되어 있는가?
- [ ] 첫 문장이 3초 안에 "왜 봐야 하는지" 답을 주는가?
- [ ] 180~250자 범위 내인가? (60초 기준)
- [ ] CTA 정확히 1개인가?
- [ ] CEO 페르소나 톤과 일치하는가?
- [ ] "안녕하세요" / "오늘은~" / "여러분" 같은 인사/예고 없는가?
- [ ] 전문용어에 "쉽게 말하면..." 풀어쓰기 있는가?
- [ ] 구체적 숫자/사례가 최소 1개 있는가?
- [ ] 본인이 봐도 "이거 터진다" 확신이 드는가?
- [ ] 검증 안 된 AI 기능·스펙 단정 0건 (불확실하면 "확인 필요" 또는 출처 표기)

모두 체크 안 되면 재작성. 체크 우회 금지.

## 검수 기준 (스킬 이밸 자동 채점)

| ID | 기준 | 합격 조건 | 가중치 |
|:---|:---|:---|:---:|
| E1 | 후킹 개수 | 3개 이상 명시 | 필수 |
| E2 | 첫 문장 후킹력 | 질문/충격/반전 중 하나로 시작 (서론·인사 없음) | 필수 |
| E3 | 글자 수 | 180~250자 (60초 기준) | 필수 |
| E4 | CTA 개수 | 정확히 1개 | 필수 |
| E5 | CEO 톤 매칭 | `context/ceo-persona.md` 규칙 준수 | 필수 |
| E6 | 전문용어 풀어쓰기 | "쉽게 말하면" 또는 동등 구조 존재 | 조건부 |
| E7 | 거창한 서론 없음 | "AI가 세상을 바꿀..." 류 0건 | 필수 |
| E8 | 구체적 사례/숫자 | 추상적 주장 대신 구체적 예시 1개+ | 권장 |
| E9 | 역순 구조 | 핵심 메시지 → 후킹 순서로 설계된 흔적 | 권장 |
| E10 | 타겟 명확성 | 대본 헤더에 구체적 타겟 명시 | 필수 |
| E11 | 대본 3버전 | 서로 다른 후킹/톤/구조의 대본 3개 | 필수 |
| E12 | 캡션 3버전 | 각 대본에 매칭되는 캡션 3개 (CTA 포함) | 필수 |
| E13 | 고객 여정 단계 | 타겟 단계 (1/2/3) 명시 완료 | 필수 |
| E14 | AI 정보 검증 | 미검증 AI 스펙 단정 0건 (출처 또는 "확인 필요") | 필수 |
| E15 | 금지행동 | `rules/금지행동.md` 9개 항목 0건 | 필수 |

**합격 기준**: 필수 항목 전체 통과 + 권장 항목 50% 이상

## 업종별 커스터마이징 팁

| 업종 | 주의할 점 |
|:---|:---|
| **1인 프리랜서** | "포트폴리오 자랑" 금지. "이 문제 해결해드려요" 포맷 |
| **카페/매장** | 비주얼 후킹 강화. 제품 샷 첫 1초 |

| **쇼핑몰** | CTA에 할인/리미티드 과장 금지. 진짜 희소성만 |
| **B2B** | 의사결정자 언어로. "대표님, ~하시죠?" 톤 |
