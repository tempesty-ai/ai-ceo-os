# 세일즈 퍼널 분석 스킬 (C.L.O.S.E 프레임워크)

> 매출이 안 나오는 진짜 원인을 5단계 퍼널로 진단.
> "왜 전환이 안 되지?"에 **구체적 병목 + 즉시 실행 액션**으로 답합니다.

---

## C.L.O.S.E 프레임워크란?

| 단계 | 의미 | 핵심 질문 |
|:---:|:---|:---|
| **C**apture | 인지/유입 | "맞는 사람이 우리를 발견했는가?" |
| **L**earn | 관심/리드 | "우리가 뭐 하는지 이해했는가?" |
| **O**ffer | 검토/제안 | "이 상품이 자기 문제 해결책으로 보이는가?" |
| **S**ettle | 전환/결제 | "지금 결제할 이유가 있는가?" |
| **E**xpand | 유지/확장 | "재구매/추천할 만한가?" |

## 2단계: 퍼널 5단계 진단

각 단계의 현재 수치와 상태를 정리:

```markdown
## 퍼널 현황 ([기간])

### [1] Capture — 인지/유입
- 채널별 방문자: (출처)
- 주요 유입 경로: (출처)
- 상태: 🟢/🟡/🔴

### [2] Learn — 관심/리드
- 리드마그넷 다운로드: (출처)
- 리드 전환율 (방문 → 리드): N% (출처)
- 상태: 🟢/🟡/🔴

### [3] Offer — 검토/제안
- 이메일 오픈율 / DM 응답률: (출처)
- 세일즈 페이지 방문 → 상담/데모 신청: N%
- 상태: 🟢/🟡/🔴

### [4] Settle — 전환/결제
- 상담 → 결제 전환율: N%
- 평균 결제 금액 (AOV): (출처)
- 상태: 🟢/🟡/🔴

### [5] Expand — 유지/확장
- 재구매율: N%
- 환불률: N%
- LTV 추세: 상승/횡보/하락
- 상태: 🟢/🟡/🔴
```

---

## 3단계: 핵심 지표 vs 위험 기준

| 지표 | 건강 | 주의 | 위험 |
|:---|:---:|:---:|:---:|
| 방문 → 리드 전환율 | 5%+ | 2~5% | <2% |
| 리드 → 상담 전환율 | 20%+ | 10~20% | <10% |
| 상담 → 결제 전환율 | 20%+ | 10~20% | <10% |
| 환불률 | <5% | 5~10% | >10% |
| LTV | 상승 | 횡보 | 하락 |

> 업종별 기준선은 다름. `knowledge/` 업계 데이터로 조정.

## 4단계: 병목 진단 (Root Cause 5 Whys)

전환율이 낮은 단계 발견 → 왜 낮은지 5번 파고들기:

```
Q1: 왜 리드 전환율이 2%인가?
A1: 리드마그넷 매력도 부족

Q2: 왜 매력도가 부족한가?
A2: 일반적인 "무료 PDF" 제목

Q3: 왜 일반적 제목을 쓰고 있는가?
A3: 타겟 고통 조사 없이 작성

Q4: 왜 조사 없이 썼나?
A4: 시간 압박 + 벤치마킹 누락

Q5: 왜 벤치마킹을 못 했나?
A5: 시스템 부재 → 근본 해결: 리드마그넷 벤치마킹 프로세스화
```

---

## 5단계: 5가지 병목 유형 진단

전환율 낮으면 아래 순서로 원인 진단:

1. **트래픽 문제?** → 유입 자체가 부족 → 콘텐츠/광고 점검
2. **리드 문제?** → 리드마그넷 매력도 부족 → 타겟 고통 재조사
3. **세일즈 메시지 문제?** → 세일즈 페이지 12-Block 구조 점검
4. **가격 문제?** → 앵커링/패키지 재설계
5. **타이밍 문제?** → 시즌/경쟁사 공격/고객 예산 사이클 점검

```markdown
2. ...

### 📅 단기 개선 (이번 달)
1. [작업] — 담당: / 기한: / 예상 효과:
2. ...
3. ...

### 🏗 구조 변경 (분기 단위, 있을 때만)

```

## 금지 사항 (절대 추천하지 말 것)

- **가짜 희소성**: "마감 임박!", "잔여 N석!" 같은 허위 긴급성
- **가짜 후기**: 만들어낸 수강생 후기, 과장된 성과
- **가격 숨기기**: 상담 강제 유도 목적의 가격 비공개
- **단기 할인 의존**: 매출 부진을 할인으로만 해결
- **타겟 확장 남용**: 병목이 퍼널 상단에 있지도 않은데 광고 예산 증액

## 세일즈 페이지 12-Block 구조 (참조)

개선안 제시 시 이 구조 기준으로:

1. Hero (후킹 메시지 + 비주얼)
2. Problem (고객 고통 명확화)
3. Outcome (달성 가능한 결과)
4. Agitate (문제 방치 시 대가)
5. Solution (우리 상품 등장)
6. Credibility (CEO/브랜드 신뢰)
7. Features (구체적 구성)
8. Social Proof (후기/사례)
9. Price (가격 + 앵커링)
10. Guarantee (환불 보장)
11. FAQ (반박 처리)
12. CTA (명확한 행동 유도)
