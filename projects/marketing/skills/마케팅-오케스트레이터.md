# 마케팅 오케스트레이터

> 마케팅 사업부 4개 에이전트(CCO·CMO·CDO·CXO)를 협업시키는 워크플로우.
> 에이전트 팀 패턴 — 에이전트들이 파일·메시지 기반으로 자체 조율한다.

---

## 실행 모드: 에이전트 팀 (기본)

4개 에이전트가 `_workspace/` 파일을 공유하며 협업한다.
단일 에이전트로 처리 가능한 작업은 이 오케스트레이터를 사용하지 않는다.

---

## Phase 0: 컨텍스트 확인

오케스트레이터 실행 시 가장 먼저 기존 산출물 존재 여부를 확인한다.

```
_workspace/ 존재 여부 확인:
  - 없음 → 초기 실행 (Phase 1부터)
  - 있음 + 부분 수정 요청 → 부분 재실행 (해당 에이전트만 재호출)
  - 있음 + 새 작업 요청 → 새 실행 (_workspace/ → _workspace_prev/ 이동)
```

---

## Phase 1: 작업 분석 및 에이전트 배정

오케스트레이터(김이사/CSO)가 CEO 요청을 분석한다.

**배정 기준:**

| 작업 유형 | 주 담당 | 크로스 렌즈 |
|:---|:---|:---|
| 릴스/캐러셀 콘텐츠 생성 | CCO | CXO (감정 정합), CMO (전환 동선) |
| 채널 성장 전략 | CMO | CDO (데이터), CXO (감정 여정) |
| 성과 분석/리포트 | CDO | CMO (전략 결정) |
| 팔로워 감정 여정 설계 | CXO | CCO (콘텐츠), CMO (퍼널) |
| 종합 마케팅 계획 | CMO + CCO | CDO, CXO |

**파일 저장 경로:** `_workspace/{phase}_{agent}_{artifact}.md`

---

## Phase 2: 병렬 실행

주 담당 에이전트와 크로스 렌즈 에이전트를 병렬로 실행한다.

```
에이전트 팀 구성:
  - 리더: CMO (전략 총괄)
  - 팀원: CCO, CDO, CXO

작업 흐름:
  1. 각 에이전트 → _workspace/에 산출물 저장
  2. 에이전트 간 크로스 리뷰 (필요 시)
  3. CMO가 종합 → 최종 산출물
```

---

## Phase 3: 통합 및 검수

CMO가 각 에이전트 산출물을 통합한다.

**통합 원칙:**
- 상충 의견: 양쪽 관점 모두 병기 + CEO 결정 필요 표시
- 데이터 없는 주장: "가설 수준" 명시
- 금지행동 최종 확인 (`rules/금지행동.md`)

**최종 산출물 저장:**
`outputs/{YYYY}/{MM}/{DD}/[전략]_{주제}_마케팅계획.md`

---

## Phase 4: CEO 브리핑

```markdown
## 마케팅팀 보고: {작업명}

**참여 에이전트**: CCO / CMO / CDO / CXO

### 핵심 산출물
(링크 또는 요약)

### CEO 결정 필요 사항
1. {항목}: [A안 / B안]

### 다음 실행 기준
- 재실행 트리거: ...
- 검증 시점: N주 후
```

---

## 에러 핸들링

| 상황 | 처리 방식 |
|:---|:---|
| 에이전트 1개 실패 | 해당 에이전트 결과 없이 진행 + 최종 보고에 "N 에이전트 산출물 누락" 명시 |
| 에이전트 간 상충 의견 | 삭제 금지. 양쪽 출처 병기 + CEO 결정 요청 |
| 데이터 없이 분석 요청 | CDO가 "데이터 필요" 차단 → 오케스트레이터가 CEO에게 데이터 요청 |
| 금지행동 위반 감지 | 전체 파이프라인 중단 + 위반 항목 보고 |

---

## 팀 크기 근거

4명(CCO·CMO·CDO·CXO)이 현 단계(팔로워 0→1만) 최적 구성이다.
CFO·CTO·COO·CPO는 수익화 단계 이후 합류.

---

## 테스트 시나리오

### 정상 흐름
```
입력: "이번 주 릴스 3개 + 성과 리뷰"
기대:
  - CCO → 릴스 3개 대본 (3버전씩)
  - CDO → 지난 주 성과 데이터 분석 요청 또는 "데이터 필요"
  - CMO → 전환 동선 점검
  - CXO → 감정 여정 정합성 검토
  출력: outputs/.../[전략]_주간마케팅.md
```

### 에러 흐름
```
입력: "지난 주 저장률 분석해줘" (데이터 없음)
기대:
  - CDO가 "데이터 필요" 응답
  - 오케스트레이터가 CEO에게 데이터 수집 방법 안내
  - 파이프라인 중단 (추정 수치 생성 금지)
```
