# 캐러셀 기획 스킬

> 마케팅 사업부 여정 **2단계 (신뢰)** 핵심 자산.
> 팔로워가 저장하고 다시 꺼내보는 "레퍼런스 콘텐츠"를 만든다.

---

## 언제 쓰나 (Trigger)

- "캐러셀 만들어줘"
- "슬라이드 N장짜리 기획해줘"
- "카드뉴스 형식으로"
- "신뢰 자산 콘텐츠 만들어줘"
- "AI 활용법 정리해서 저장용으로"

---

## 자동 로드할 것

- `projects/marketing/CLAUDE.md` (사업부 헌법)

- `projects/marketing/context/channel-profile.md` (채널 프로필)

## 캐러셀 구조 원칙

### 슬라이드 역할 분배

| 슬라이드 | 역할 | 원칙 |
|:---|:---|:---|
| **1장 (커버)** | 후킹 — 저장 이유 | "이걸 저장하면 뭘 얻는가" 즉답 |
| **2~N장 (본문)** | 핵심 인사이트 전달 | 슬라이드당 1개 메시지. 복잡하면 분리 |
| **마지막 장 (CTA)** | 행동 유도 | 저장/팔로우/댓글 중 1개만 |

### 슬라이드 수 가이드

| 주제 유형 | 권장 슬라이드 수 |
|:---|:---|
| 팁 목록 (5가지, 7가지 등) | 주제 수 + 커버 + CTA |
| 단계별 설명 (How-to) | 단계 수 + 커버 + CTA |
| 비교 (전후, A vs B) | 4~6장 |
| 심층 인사이트 | 8~12장 이내 |

---

## 3-Phase 파이프라인

### Phase 1. 기획

- 타겟 여정 단계 명시 (주로 2단계)
- 저장 이유 1문장: "팔로워가 이걸 저장하는 이유는 ___이다"
- 슬라이드 목차 (1장당 제목 1개)

### Phase 2. 원고 작성

각 슬라이드별 원고:
- **헤드라인** (굵게 보일 텍스트 — 10자 이내 권장)
- **서브카피** (헤드라인을 보완하는 1~2문장)
- **비주얼 가이드** (이미지/아이콘/배색 방향, 선택)

### Phase 3. 검수

- [ ] 1장 커버에 저장 이유가 있는가
- [ ] 슬라이드당 메시지 1개 원칙 지켜졌는가
- [ ] 마지막 장 CTA 1개만 있는가
- [ ] 금지행동 위반 0건
- [ ] CEO 톤앤매너 일치
