# 통합 오케스트레이터

> `projects/` 하위 모든 사업부를 조율하는 컨트롤 타워.
> 새 사업부가 추가되면 이 파일의 **사업부 레지스트리**에만 등록하면 자동 적용.

---

## 사업부 레지스트리

> 새 사업부 신설 시 이 테이블에 1줄 추가.

| 사업부 | 경로 | 미션 한 줄 | 핵심 KPI |
|:---|:---|:---|:---|
| marketing | `projects/marketing/` | 팔로워 전환 (여정 1~3단계) | 팔로우 전환율, 저장률 |
| contents | `projects/contents/` | 조회수·바이럴·인기 | 시청 완료율, 공유율 |

---

## 협업 트리거 (언제 이 오케스트레이터를 쓰는가)

단일 사업부로 해결 가능 → 해당 사업부 스킬 직접 사용.
아래 조건 중 하나라도 해당 → 통합 오케스트레이터 가동.

- [ ] 2개 이상 사업부의 KPI가 동시에 관련된 작업
- [ ] 콘텐츠 생산(contents) + 팔로우 전환(marketing) 동시 최적화
- [ ] 주간/월간 통합 콘텐츠 캘린더
- [ ] 크로스 사업부 성과 분석
- [ ] 신규 사업부 온보딩

---

## 협업 프로토콜

### Phase 0: 컨텍스트 확인
```
기존 _workspace/ 있음 + 부분 수정 요청 → 해당 사업부만 재실행
기존 _workspace/ 있음 + 새 작업 → _workspace_prev/ 이동 후 새 실행
_workspace/ 없음 → 초기 실행
```

### Phase 1: 작업 분류 및 배분
1. 요청을 읽고 관련 사업부 식별 (레지스트리 참조)
2. 각 사업부 CLAUDE.md를 읽어 KPI·미션 확인
3. 작업을 사업부별로 분리 배분

### Phase 2: 사업부별 실행
각 사업부가 담당 스킬로 독립 실행.
산출물은 `_workspace/{사업부명}_{산출물}.md`에 저장.

### Phase 3: 교차 검수
- **contents 산출물 → marketing 검수**: "이 콘텐츠가 팔로우 전환으로 이어지는가?"
- **marketing 산출물 → contents 검수**: "이 전환 전략이 조회수를 희생시키지 않는가?"
- 충돌 발생 시 → CSO 김이사 중재

### Phase 4: 통합 산출물 생성
두 사업부 관점이 모두 반영된 최종 결과물 출력.

## 핵심 협업 접점

### 접점 1: 콘텐츠 생산 파이프라인
```
contents CCO (바이럴 기획·3버전)
    ↓
contents CDO (성과 예측·3신호 체크)
    ↓
marketing CCO (팔로우 전환 관점 검수)
    ↓
marketing CMO (플랫폼 최적화·타이밍)
    ↓
업로드
```

### 접점 2: 주간 통합 캘린더
```
contents (멀티플랫폼-캘린더 스킬) → 조회수 최적화 배치
    ↔ 교차 검토
marketing (콘텐츠 믹스) → 고객 여정 단계별 배분
    ↓
통합 주간 캘린더 (조회수 + 전환 동시 최적화)
```

### 접점 3: 성과 분석
```
contents CDO: 조회수·시청 완료율·공유율
    +
marketing CDO: 팔로우 전환율·저장률·DM 문의
    ↓
통합 인사이트: "조회수는 높은데 팔로우 전환이 낮다" 류 패턴 발견
```

## 충돌 해결 원칙

두 사업부 의견이 충돌할 때:

| 상황 | 우선순위 |
|:---|:---|
| 채널 초기 성장 단계 (현재) | contents 우선 — 조회수·도달이 먼저 |
| 팔로워 1천명 이후 | marketing 우선 — 전환 최적화 |
| 항상 | 품질 희생 없이 조율 가능하면 양쪽 반영 |

## 신규 사업부 온보딩 체크리스트

새 사업부(`projects/{name}/`)가 생기면:

- [ ] 레지스트리 테이블에 1줄 추가
- [ ] 해당 사업부 CLAUDE.md에 협업 프로토콜 섹션 추가
- [ ] 기존 사업부 CLAUDE.md의 협업 대상에 신규 사업부 추가
- [ ] 접점 테이블에 새 협업 접점 정의
