# scripts/ — 운영 자동화 스크립트

> 백업, 알림, 동기화 등 인프라 운영 자동화 스크립트가 들어가는 폴더입니다.

## 구조 (예시)

```
scripts/
├── git_backup.sh        ← GitHub 백업 자동화
├── discord_notify.sh    ← 디스코드 알림 발송
├── daily_briefing.sh    ← 매일 아침 브리핑 자동 발송
└── crontab_master.txt   ← 크론 스케줄 마스터 파일
```

## 특징

- **execution/** 과 차이: execution은 비즈니스 로직(매출 분석, 콘텐츠 수집), scripts는 인프라 운영(백업, 알림, 배포)
- 처음 시작 시 비어있어도 OK. 필요할 때 추가하면 됨

## 김이사가 처음 셋업하는 자동화 (추천 순서)

1. **GitHub 백업** — 매일 1회 자동 푸시 (코드/문서 안전)
2. **Discord 알림** — 작업 완료 시 알림
3. **데이터 동기화** — Google Sheets, Notion 등과 자동 동기화
4. **크론 스케줄** — 정기 작업 자동 실행

## 사용 시점

CEO가 "이거 매일 자동으로 돌리고 싶어"라고 하면 김이사가 이 폴더에 스크립트를 만듭니다.
