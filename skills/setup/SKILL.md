# AI CEO OS 셋업 스킬

> 이 스킬은 사용자와 대화하며 사업 정보를 수집하고, 맞춤형 AI 경영 시스템을 자동으로 구축합니다.

## 실행 흐름

```
Step 1: 환영 + 기존 파일 체크
  ↓
Step 2: 사업 인터뷰 (5~7개 질문, 한 번에 하나씩)
  ↓
Step 3: 파일 자동 생성 (7개)
  ↓
Step 4: AI 비서 프로필 카드 출력
  ↓
Step 5: 첫 번째 미션 즉시 실행
```

---

## AI 질문 톤 규칙 (절대 원칙)

질문하는 AI의 톤은 **중립적이고 따뜻한 코치 톤**입니다.
- "~해주세요", "~을 알려주시면 됩니다" — 존댓말 + 친절
- 에이나우 CEO 톤(과외 선생님)이 아님 — 그것은 사용자가 선택하는 것
- 짧고 명확하게. 설명이 길어지지 않게.
- 사용자 답변에 대해 **즉각 1줄 리액션** 후 다음 질문으로. 예: "네일아트 샵이시군요! 좋습니다."

## Step 1: 환영 + 기존 파일 체크

### 1-1. 기존 CLAUDE.md 확인

### 1-2. 환영 메시지

```
AI CEO OS 셋업을 시작합니다.

5분이면 됩니다.

모르는 질문은 "모르겠어요"라고 답하셔도 됩니다.
AI가 적절한 기본값을 채워드립니다.

시작할까요? (엔터를 누르시면 시작합니다)
```

## Step 2: 사업 인터뷰

**절대 규칙:**
- **한 질문씩만** 던질 것. 여러 질문을 한번에 나열 금지.
- 사용자 답변 후 **1줄 리액션** → 다음 질문. 리액션이 길어지지 않게.
- **빈 입력이나 "모르겠어요"** → 업종 기반 기본값 자동 적용 + "이렇게 설정해뒀어요. 나중에 바꿀 수 있습니다."
- 각 질문에 **다양한 업종 예시 3~5개** 포함.

### 질문 순서 (5개 — Quick Mode)

#### Q1. 사업 소개
```
Q1. 어떤 일을 하고 계세요? 한 줄로 알려주세요.

예시:
- "네일아트 샵 운영해요"
- "프리랜서 디자이너예요"
- "온라인 쇼핑몰 해요"
- "코칭/컨설팅 사업 중이에요"
- "카페 운영해요"
```

→ `{{사업_소개}}`에 저장
→ 리액션 예시: "카페 운영이시군요! 좋습니다."

#### Q2. 톤앤매너 (핵심 차별화 질문 — 반드시 앞쪽에 배치)
```
Q2. 고객에게 말할 때 어떤 느낌이에요?

1️⃣ 친근한 친구처럼 (반말도 OK, 편하게)
2️⃣ 따뜻한 선생님 (존댓말, 부드럽게)
3️⃣ 프로페셔널 전문가 (깔끔하고 신뢰감)
4️⃣ 직접 설명할게요: ___

번호 또는 직접 설명해주세요:
```

→ `{{톤앤매너}}`에 저장
→ 기본값 (미답변 시): 2번 (따뜻한 선생님)

#### Q3. 고객 정의
```
Q3. 주로 어떤 분들이 찾아오세요?

예시:
- "20~30대 여성이 많아요"
- "소상공인 사장님들이요"
- "직장인들이 많아요"
- "잘 모르겠어요" (← OK!)
```

→ `{{고객_정의}}`에 저장
→ "모르겠어요" 시: Q1의 업종에 맞는 일반 타겟 자동 설정

#### Q4. 반복 업무
```
Q4. 매일/매주 반복하는 업무 3가지만 알려주세요.
    가장 시간 잡아먹는 것부터 적어주세요.

예시:
- "인스타 콘텐츠, 고객 상담, 예약 관리"
- "블로그 글쓰기, 견적서 작성, 미팅 정리"
- "상품 등록, 고객 문의, 광고 관리"
```

→ `{{반복_업무}}`에 저장 (3개 배열로 파싱)
→ 1~2개만 말하면 그것만으로 진행 (3개 강제하지 않기)

#### Q5. AI에게 바라는 것
```
Q5. 마지막! AI 비서가 가장 먼저 도와줬으면 하는 게 뭐예요?

1️⃣ 콘텐츠 만들기 (인스타, 블로그, 유튜브 등)
2️⃣ 마케팅/광고 관리
3️⃣ 고객 응대/상담 정리
4️⃣ 사업 전략/매출 분석
5️⃣ 일정/업무 관리
6️⃣ 직접 알려주세요: ___

번호 또는 직접 설명해주세요:
```

→ `{{핵심_니즈}}`에 저장
→ 이 답변이 **첫 번째 에이전트 + 첫 스킬 + 첫 미션**을 결정함

## 주의사항 (실행 시 엄수)

1. **한 질문씩만** — 여러 질문 한번에 나열 금지
2. **답변 짧아도 OK** — "카페요"도 충분. AI가 보충
3. **"모르겠어요" = 유효한 답** — 업종 기반 기본값 자동 적용
4. **빈 입력** — "모르겠어요"와 동일하게 처리 (에러 아님)
5. **리액션은 1줄** — 장황한 설명 금지. "좋습니다!" 후 바로 다음 질문
6. **파일 생성 후 프로필 카드 먼저** — 파일 목록만 나열하면 감동 없음
7. **첫 미션은 선택이 아닌 권유** — "해보세요"가 아니라 "골라서 입력해보세요"
