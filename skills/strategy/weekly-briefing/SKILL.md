# 주간 브리핑 스킬

> 매주 월요일(또는 CEO 요청 시) 5분 안에 사업 전체 상태를 보고.
> CSO 김이사 서브에이전트가 실행하는 표준 브리핑 프로토콜.

---

## 언제 쓰는가

- 매주 월요일 아침 자동 실행 (크론 연동 시)
- CEO가 "브리핑 줘" / "이번 주 어때" 요청 시
- 주요 결정 직전 "현황 스냅샷 필요" 시

**금지**: 바빠서 "간단히 요약해줘" 요청 시에도 5가지 필수 섹션을 전부 포함할 것. 축약 금지.

---

## 0단계: 김이사 모드 활성화

`.claude/agents/cso.md` 서브에이전트로 전환. (이미 활성화 상태면 생략)

---

## 1단계: 자산 병렬 로드 (Phase 1)

아래 파일을 **동시에** 읽습니다:

### 필수
- `tasks/session_handoff.md` — 이전 세션 맥락
- `tasks/todo.md` — 현재 진행 작업
- `tasks/lessons.md` — 최근 교훈 (클러스터별 패턴)
- `tasks/predictions.md` — 검증 기한 도래 예측

### 조건부 (있으면)
- `CLAUDE.md` — 사업 맥락
- `context/mission-vision.md` — 전략 테제
- `knowledge/channel-data/` 하위 최신 성과 데이터
- `execution/` 스크립트 결과물 (있으면)

---

## 2단계: 시스템 위생 점검 (Garbage Collection)

브리핑 **작성 전** 아래 점검을 수행. 이상 발견 시 브리핑 하단 "시스템 위생" 섹션으로 보고.

### 점검 항목 5개

| # | 점검 대상 | 검증 방법 | 이상 기준 |
|:---|:---|:---|:---|
| 1 | `tasks/todo.md` 정체 작업 | 진행 중 항목의 갱신 시점 확인 | 7일+ 정체 |
| 2 | `tasks/lessons.md` 기록 주기 | 마지막 교훈 날짜 확인 | 14일+ 기록 없음 (학습 루프 정체) |
| 3 | `tasks/predictions.md` 검증 지연 | 검증일 지난 예측 | 검증 기한 초과 1건+ |
| 4 | `tasks/session_handoff.md` 최신성 | 마지막 업데이트 시점 | 7일+ 미업데이트 |
| 5 | 파일 참조 정합성 | Grep으로 주요 경로 깨짐 확인 | 깨진 경로 1건+ |

### 이상 없을 시
브리핑 말미에 한 줄: `시스템 위생: 정상 ✓`

### 이상 발견 시

```markdown
## 🔧 시스템 위생 [N건 이상]
| # | 이상 항목 | 내용 | 조치 제안 |
|:---|:---|:---|:---|
| 1 | [항목] | [구체적 내용] | [수정 방법] |
```

## 3단계: 브리핑 작성 (Phase 2)

### 작동 원칙
- **위험 신호 우선**: 🔴 위험 항목을 맨 위에 배치
- **5분 분량**: A4 2페이지 이내. 불필요한 서술 금지, 표와 핵심만
- **Zero-Inference**: 숫자 직접 계산 금지. 모든 수치에 출처 명시
- **선택지 제시**: CEO 판단 요청 시 반드시 **2개 이상 선택지** + 김이사 추천안

### 표준 보고 템플릿

```markdown
# 주간 브리핑 — [YYYY-MM-DD]
> 김이사 제공. 5분 안에 읽는 전사 현황.

## 🔴 위험 신호 (최우선)
| # | 프로젝트/영역 | 상태 | 원인 | 조치 제안 | 확신도 |
|:---|:---|:---|:---|:---|:---:|
| 1 | ... | 🔴 지연 | ... | ... | N% |

**이상 없음 시**: "위험 신호 없음. 🟢"

## 📊 핵심 지표 (지난 주)
| 지표 | 이번 주 | 전주 | 증감 | 목표 대비 |
|:---|:---:|:---:|:---:|:---:|
| 매출 | (출처) | | | |
| 리드 | (출처) | | | |
| 전환율 | (출처) | | | |
| 콘텐츠 도달 | (출처) | | | |

> 모든 수치에 출처 명시. 출처 없으면 "데이터 필요"로 표기.

## ✅ 지난 주 완료
- [완료 1]
- [완료 2]
- [완료 3]

## ⏳ 진행 중 (이번 주 마감 임박)
| 작업 | 담당 | 기한 | 상태 | 블로커 |
|:---|:---|:---:|:---:|:---|
| ... | ... | D-N | 🟡 | (있으면) |

## 🎯 이번 주 김이사 추천 집중 Top 3
1. **[최우선]** [작업] — 이유: [왜 이게 Top 1인지] (확신도: N%)
2. **[차순위]** [작업] — 이유: ... (확신도: N%)
3. **[병행]** [작업] — 이유: ... (확신도: N%)

## ⚠️ CEO 판단 요청 (있으면)
### 안건 1: [안건명]
- **옵션 A**: [설명] — 장점: / 단점: / 확신도: N%
- **옵션 B**: [설명] — 장점: / 단점: / 확신도: N%
- **김이사 추천**: [A 또는 B] — 이유: [3줄 이내]

## 📡 선제 시그널 (CEO가 짚기 전에 김이사가 발견)
| # | 시그널 | 긴급도 | 제안 액션 |
|:---|:---|:---:|:---|
| 1 | ... | 🔴/🟡/🟢 | ... |

## 🔧 시스템 위생
[정상 ✓ 또는 이상 N건 표]

## 📈 예측 검증 (predictions.md 기한 도래 항목)
- [P-XXX] [예측 제목] — 결과: (적중/빗나감/폐기) — 오차 분석: ...
- 새 예측 등록 제안: (있으면)
```

## 4단계: 검증 (Phase 3)

셀프 체크리스트:

- [ ] 🔴 위험 신호가 최상단에 있는가?
- [ ] A4 2페이지 이내인가?
- [ ] 모든 수치에 출처가 명시되어 있는가?
- [ ] CEO 판단 요청에 **2개 이상 선택지 + 김이사 추천안**이 있는가?
- [ ] 시스템 위생 점검 5가지 수행했는가?
- [ ] 김이사 추천 Top 3 각각에 **확신도(%)**가 표기되었는가?
- [ ] 지난 주 완료 / 진행 중 / 이번 주 집중이 분리되어 있는가?
- [ ] 선제 시그널 최소 1건은 있는가? (없으면 "선제 시그널 없음" 명시)

## 검수 기준

| ID | 기준 | 합격 조건 | 가중치 |
|:---|:---|:---|:---:|
| E1 | 위험 신호 최상단 | 🔴 항목이 보고서 최상단 | 필수 |
| E2 | 5분 분량 | A4 2페이지 이내 | 필수 |
| E3 | 선택지 2개+ | 판단 요청 시 2개 이상 옵션 | 필수 |
| E4 | 시스템 위생 점검 | 5가지 점검 결과 포함 | 필수 |
| E5 | Zero-Inference | 숫자 직접 계산 없음, 출처 명시 | 필수 |
| E6 | 확신도 표기 | 모든 추천에 % 명시 | 필수 |
| E7 | 마감 감지 | D-7 이내 마감 항목 별도 표시 | 권장 |
| E8 | 정체 작업 식별 | 7일+ 미진행 작업 플래그 | 권장 |
| E9 | 선제 시그널 | 최소 1건 보고 또는 "없음" 명시 | 권장 |

**합격 기준**: 필수 항목 전체 통과
