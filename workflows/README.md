# Workflows (파이프라인)

> 여러 에이전트가 협업하는 자동화 파이프라인입니다.
> 1인 기업이 팀처럼 돌아가는 핵심 구조.

---

## 파이프라인 목록

### 콘텐츠
| 파이프라인 | 목적 | 흐름 |
|:---|:---|:---|
| 릴스 대본 | 후킹+바이럴 공식 적용 대본 생성 | 리서치 → 초안 → 검증 |
| 유튜브 기획 | 영상 기획부터 대본 완성 | 트렌드 분석 → 기획 → 대본 |
| 콘텐츠 캘린더 | 월간 콘텐츠 계획 수립 | 트렌드 → 전략 → 배분 |
| 콘텐츠 재활용 | 1개 영상 → 4채널 콘텐츠 | 변환 → 최적화 → 배포 |
| 자율 콘텐츠 팩토리 | 콘텐츠 자동 생산 시스템 | 감지 → 생산 → 배포 |

### 마케팅/세일즈
| 파이프라인 | 목적 | 흐름 |
|:---|:---|:---|
| 마케팅 캠페인 분석 | ROAS/CPA 성과 분석 | 데이터 수집 → 분석 → 최적화 |
| 세일즈 퍼널 분석 | 전환율 최적화 | 퍼널 진단 → 병목 발견 → 개선 |
| 콘텐츠→매출 전환 | 무료 콘텐츠 → 유료 전환 | 유입 → 너처링 → 전환 |
| 시장 분석 | 경쟁사 벤치마킹 | 리서치 → 분석 → 전략 |

### 경영/전략
| 파이프라인 | 목적 | 흐름 |
|:---|:---|:---|
| 주간 브리핑 | 전사 현황 5분 보고 | 데이터 수집 → 종합 → 보고 |
| 월간 사업 리뷰 | OKR 점검 + P&L | 성과 수집 → 분석 → 조정 |
| B2B 제안서 | 고객사 맞춤 제안서 | 리서치 → 작성 → 검증 |

### 제품/교육
| 파이프라인 | 목적 | 흐름 |
|:---|:---|:---|
| 강의 설계 | 커리큘럼 설계 | 분석 → 설계 → 검증 |
| 강의 성과 분석 | 완강률/NPS 분석 | 데이터 → 인사이트 → 개선 |
| 신규 강의 출시 | 기획부터 런칭까지 | 기획 → 제작 → 마케팅 → 런칭 |
| SaaS MVP 출시 | 검증부터 출시까지 | PMF → MVP → 런칭 |

### 기타

|:---|:---|:---|

## 3-Phase 파이프라인 구조 (권장)

모든 파이프라인은 이 구조를 따릅니다:

```
Phase 1: 리서치 + 자산 로딩 (병렬)
├── 메인 에이전트: 전사 자산 로드
├── 서브에이전트 A: 웹 리서치
└── 서브에이전트 B: 내부 데이터 분석
         ↓
Phase 2: 통합 + 초안 작성
└── 메인 에이전트: 리서치 결과 + 자산 → 초안
         ↓
Phase 3: 검증 (병렬)
├── 검증 에이전트 A: 관점 1에서 검토
└── 검증 에이전트 B: 관점 2에서 검토
         ↓
최종 수정 + 납품
```
