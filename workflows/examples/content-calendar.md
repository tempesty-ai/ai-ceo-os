# 콘텐츠 캘린더 파이프라인 (Gold Standard)

> 다음 달(또는 지정 기간) 콘텐츠 계획을 자동 수립.
> CEO가 "이번 달 뭘 올리지?" 고민하지 않게 하는 파이프라인.

---

## CEO가 치는 한 줄

```
김이사, 5월 콘텐츠 캘린더 짜줘.
```

또는 채널 지정:

```
김이사, 5월 인스타 캘린더만 주 5회 기준으로.
```

---

## Phase 1 — 병렬 자산 로딩

```markdown
Read 병렬:
- CLAUDE.md (고객 여정 6단계, 자주 시키는 작업)
- context/ceo-persona.md (톤앤매너)
- context/channel-rules.md (채널별 발행 주기, 해시태그, CTA 규칙)
- tasks/todo.md (이번 달 프로모션/런칭 일정)
- knowledge/channel-data/ (지난 달 성과 데이터, 있으면)
```

---

## Phase 2 — Tier 2 병렬 소집 (CCO + CMO 필수)

```python
# 동시 호출 (병렬)

Task(
    subagent_type="cco",
    description="콘텐츠 성과 분석 + 주제 발산",
    prompt="""
    [입력 데이터]
    - 지난 30일 콘텐츠 성과 (knowledge/channel-data/)
    - CEO 페르소나 및 채널 규칙
    - 이번 달 사업 목표 (CLAUDE.md 고객 여정 단계별)

    [요청 산출물]
    1. 지난 달 TOP 3 콘텐츠 + 바이럴 요인 분석
    2. 지난 달 BOTTOM 3 콘텐츠 + 실패 요인
    3. 이번 달 시도할 새 포맷 2개 (근거 포함)
    4. 이번 달 발산 주제 20개 (유형별 분류: 교육/트렌드/스토리/논쟁/재활용)
    5. Cross-Lens (CMO/CXO 관점 영향 1줄씩)
    6. Inversion (망치는 3가지 시나리오)

    Zero-Inference 엄수. 수치는 출처 명시.
    """
)

Task(
    subagent_type="cmo",
    description="이번 달 프로모션/세일즈 일정 + 전환 목표",
    prompt="""
    [요청 산출물]
    1. 이번 달 프로모션 일정 (런칭/세일/웨비나/라이브 등)
    2. 각 프로모션에 선행되어야 할 "빌드업 콘텐츠" 유형
    3. CTA 전략 (도달 확장 vs 리드 수집 vs 구매 전환의 주간 비중)
    4. 콘텐츠 → 매출 전환 경로 1줄 정의
    5. Cross-Lens + Inversion
    """
)
```

## Phase 3 — 김이사 종합 + 캘린더 작성

각 임원의 독립 보고를 김이사가 종합하여 아래 구조로 작성:

### 1. 월간 테마 확정
```markdown
## [YYYY년 MM월] 월간 테마
**테마**: [1문장 — 이번 달 이야기할 메인 서사]
**이유**: [사업 목표 + CCO 주제 발산 + CMO 프로모션 일정 종합]
**핵심 KPI**: [도달 N / 리드 N / 전환 N]
```

### 2. 주간 배분 구조

| 요일 | 인스타 | 유튜브 | 뉴스레터 | 목적 |
|:---:|:---|:---|:---|:---|
| 월 | 교육형 릴스 | — | — | 가치 전달 |
| 화 | 트렌드 릴스 | — | — | 도달 확장 |
| 수 | 카드뉴스 | 롱폼 1편 | — | 신뢰 구축 |
| 목 | 교육형 릴스 | — | 주간 레터 | 가치 + 관계 |
| 금 | 바이럴 릴스 | 쇼츠 | — | 도달 확장 |
| 토 | 스토리텔링 | — | — | 감정 연결 |
| 일 | 재활용 | — | — | 리소스 효율 |

### 3. 일별 콘텐츠 카드

```markdown
## Week 1

### 5/5 (월) — 인스타 릴스
- **유형**: 교육형
- **주제**: [구체적 제목]
- **후킹**: [3개 후킹 아이디어]
- **CTA**: [1개 — 채널 규칙 준수]
- **고객 여정**: 2단계 (신뢰 구축)
- **KPI 기여**: 도달 확장
- **제작 메모**: [필요한 촬영/자료]

### 5/5 (월) — 유튜브 롱폼 (격주)
- ...
```

## Phase 4 — 검증 (Cross-Lens)

캘린더 작성 후 서브에이전트 재호출로 검증:

```python
Task(
    subagent_type="cco",
    description="캘린더 브랜딩 정합 검증",
    prompt="""
    아래 캘린더를 검토:
    [캘린더 전문]

    체크:
    1. CEO 톤앤매너 일관성
    2. 채널별 발행 주기 준수
    3. 재활용 콘텐츠 품질 (단순 복붙 아닌가)
    4. 브랜드 테마와의 정합성
    
    이슈 발견 시 수정 제안 3개 이내로 반환.
    """
)

Task(
    subagent_type="cmo",
    description="캘린더 전환 퍼널 정합 검증",
    prompt="""
    이 캘린더가:
    1. 이번 달 프로모션 일정과 시점이 맞는가
    2. 고객 여정 단계별 비율이 적정한가 (상단/중단/하단 퍼널)
    3. CTA가 너무 팔기만 하거나, 너무 가치만 주는 편향이 없는가
    
    이슈 발견 시 수정 제안 3개 이내.
    """
)
```

## 품질 기준

- [ ] 사업 목표와 연결된 월간 테마
- [ ] 프로모션 일정 반영
- [ ] 일별 콘텐츠 카드 (제목 뿐 아니라 후킹+CTA+KPI 기여 포함)
- [ ] CCO + CMO 검증 완료
- [ ] Cross-Lens + Inversion 포함
- [ ] Zero-Inference (지난 달 성과 인용 시 출처)

---

## 이 파이프라인이 실패하는 3가지 시나리오

1. **재활용 콘텐츠 남용 → 브랜드 피로도** → 조기 경고: 재활용 비율 20% 이하 유지
2. **프로모션 빌드업 부재 → 갑작스러운 세일즈 메시지** → 조기 경고: 프로모션 D-7부터 관련 콘텐츠 배치
3. **주제 발산 20개 중 현실 제작 가능 5개 미만** → 조기 경고: 제작 리소스 역산 후 현실적 개수 확정
