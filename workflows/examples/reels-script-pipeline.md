# 릴스 대본 파이프라인 (Gold Standard)

> 주제 입력 → 대본 3버전 + 캡션 3버전 산출.
> CCO 서브에이전트 주도, CMO가 전환 관점에서 검증.

---

## CEO가 치는 한 줄

```
김이사, AI 초보자가 챗GPT로 시간 단축하는 주제로 릴스 대본 써줘.
```

또는:

```
/reels-script AI 초보자 챗GPT 시간단축
```

---

## Phase 1 — 병렬 자산 로딩

```markdown
Read 병렬:
- CLAUDE.md (고객 여정, 자주 시키는 작업)
- context/ceo-persona.md (톤앤매너 — 가장 중요)
- context/channel-rules.md (해시태그, CTA 규칙)
- skills/content/reels-script/SKILL.md (스킬 본문 — 후킹 12타입, 품질 기준)
- tasks/lessons.md (콘텐츠 관련 과거 교훈)

조건부:
- knowledge/channel-data/ (본인 릴스 인사이트)
- 실제 콘텐츠 원본 (있으면 톤/패턴 학습용)
```

---

## Phase 2 — CCO 서브에이전트 소집

```python
Task(
    subagent_type="cco",
    description="릴스 대본 3버전 + 캡션 3버전 초안",
    prompt="""
    주제: {user_topic}
    타겟: (CLAUDE.md 고객 여정 단계 기준)
    길이: 30~60초

    [요구사항]
    1. 역순 대본 작성법 (핵심 메시지 → 후킹 설계)
    2. 후킹 3개 이상 명시 (skills/content/reels-script/SKILL.md의 12타입 참조)
    3. 서로 다른 구조/톤의 대본 3버전 (복붙 변형 금지)
    4. 각 대본에 매칭되는 캡션 3버전 (CTA 정확히 1개)
    5. 180~250자 범위 (60초 기준)
    6. CEO 페르소나 100% 준수
    7. "안녕하세요/오늘은/여러분" 인사 금지
    8. 전문용어 쓸 때 "쉽게 말하면..." 풀어쓰기

    [출력 형식]
    skills/content/reels-script/SKILL.md의 5단계 출력 형식 그대로.

    [Cross-Lens 필수]
    - CMO: (전환 관점 영향 1줄)
    - CXO: (팬덤 형성 영향 1줄)

    [Inversion 필수]
    이 대본이 안 터지는 3가지 시나리오 + 조기 경고.
    """
)
```

---

## Phase 3 — CMO 전환 검증 소집 (선택)

콘텐츠 목적이 "도달 확장"만이면 생략 가능. "리드/전환" 목적이면 필수.

```python
Task(
    subagent_type="cmo",
    description="대본 3버전 전환 관점 검증",
    prompt="""
    [대본 3버전]
    {cco_output}

    체크:
    1. CTA가 정확히 1개이고, 고객 여정의 어느 단계를 타겟하는가
    2. 구매 전환 유도 vs 리드 수집 vs 도달 확장 중 어디에 기울어 있는가
    3. 과장/과대 약속 표현이 없는가 ("이거 하나면 월 1000만원!" 류)
    4. 해시태그 및 캡션 CTA가 채널 규칙과 일치하는가

    3개 대본 각각에 "전환 기여도 점수 (1~5)" + 개선 제안 1개.
    """
)
```

---

## Phase 4 — 김이사 종합 + 셀프 검증

CCO + CMO 응답을 김이사가 종합:

1. **충돌 조정**: CCO "후킹 강화" vs CMO "전환 유도 강화" 충돌 시 → **고객 여정 단계가 우선 기준**. 도달 확장 단계면 CCO, 전환 단계면 CMO 편.
2. **최종 3버전 확정**: CMO 전환 기여도 점수 + CCO 바이럴 예측 종합
3. **품질 체크리스트 E1~E12 전부 통과 확인**

### 셀프 검증 체크리스트
- [ ] 후킹 3개 이상?
- [ ] 첫 문장 3초 후킹?
- [ ] 글자 수 180~250자?
- [ ] CTA 정확히 1개?
- [ ] CEO 톤 일치?
- [ ] 인사/예고 없음?
- [ ] 전문용어 풀어쓰기?
- [ ] 구체적 숫자/사례?
- [ ] 타겟 명확?
- [ ] 대본 3버전 + 캡션 3버전?

통과 못 하면 CCO 재호출 1회.

```markdown
- 타겟:
- 고객 여정 단계: N
- KPI 목적: 도달 확장 / 리드 수집 / 구매 전환
- 예상 발행일:

(skills/content/reels-script/SKILL.md 5단계 형식)

## 캡션 3버전
(동일 형식)

## 김이사 판단
- 추천 Top 1: 대본 N — 이유: (확신도: N%)
- 이 대본 사용 시 예측 기록: predictions.md에 P-XXX로 등록 제안
```

## 이 파이프라인이 실패하는 3가지 시나리오 (Inversion)

1. **3버전이 "복붙 변형"** → 조기 경고: 후킹 타입이 겹치면 재작성 강제
2. **CEO 톤 이탈** → 조기 경고: `context/ceo-persona.md` 금지 표현 1개라도 섞이면 재작성
3. **Phase 3 (CMO 검증) 스킵** → 조기 경고: 전환 목적 콘텐츠인데 CMO 검증 없으면 경고 배너
