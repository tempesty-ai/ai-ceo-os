# 🥇 주간 브리핑 파이프라인 (Gold Standard)

> **이 문서는 "실제로 돌아가는" 참조 구현(reference implementation)입니다.**
> 다른 모든 워크플로우를 이 형식에 맞춰 재작성하세요.

매주 월요일, 전사 현황을 5분 안에 CEO에게 보고하는 완전 자동 파이프라인.

---

## 언제 실행되는가

1. **CEO 수동**: `/weekly-briefing` 또는 "김이사 브리핑 줘"
2. **크론 자동**: 매주 월요일 오전 8시 (`scripts/weekly_briefing_cron.sh` 설정 시)
3. **Claude Code hook**: 세션 시작 시 7일 이상 경과 감지하면 자동 발동

## Phase 1 — 병렬 자산 로딩 (김이사 단독)

김이사(cso 서브에이전트)가 아래 파일을 **동시에** 로드:

```markdown
Read tool 병렬 호출:
- tasks/session_handoff.md
- tasks/todo.md
- tasks/lessons.md
- tasks/predictions.md
- CLAUDE.md
- context/mission-vision.md
```

**조건부 병렬 로드** (파일 있으면):
```markdown
- knowledge/channel-data/*.md (최근 성과 데이터)
- execution/ 스크립트 실행 결과 (KPI 집계)
```

## Phase 2 — Tier 2 판단 (필요 시 서브에이전트 병렬 소집)

### 소집 결정 로직

| 조건 | 소집 임원 |
|:---|:---|
| 지난 주 광고 집행 있음 | **CMO + CDO** |
| 이번 달 매출 목표 70% 미달 | **CFO + CMO** |
| 부트캠프/강의 운영 중 | **CPO + CXO** |
| 콘텐츠 발행 있음 | **CCO** |
| 위 조건 모두 해당 없음 | **김이사 단독 (Tier 1)** |

### Tier 2 소집 시 Task 호출 블록 (실제 실행 코드)

```python
# Claude Code Task tool을 병렬 호출
# 각 임원이 독립 컨텍스트로 데이터 분석 후 Raw Report 반환

Task(
    subagent_type="cmo",
    description="마케팅 주간 리포트",
    prompt="""
    지난 주(월~일) 마케팅 성과를 아래 형식으로 보고하세요:

    1. 광고 성과 (ROAS/CPA/지출/매출) — Zero-Inference, 출처 명시
    2. 콘텐츠 → 세일즈 전환 퍼널 현황
    3. 전주 대비 주요 변화 3건
    4. 이번 주 1순위 개선안 + 확신도
    5. Cross-Lens: CFO/CDO 관점 영향 1줄씩
    6. Inversion: 이 주간 계획을 망치는 3가지 시나리오
    
    길이: 표 중심, 500자 이내.
    """
)

# 병렬로 동시에:
Task(
    subagent_type="cdo",
    description="데이터 주간 리포트",
    prompt="""
    지난 주 KPI 데이터 점검:
    1. 표본 크기와 통계적 유의성 평가
    2. 전주 대비 이상치 감지 (±20% 초과)
    3. 대리 지표(vanity metric) 경고
    4. 측정 누락된 지표 플래그
    Zero-Inference 엄수. 데이터 없으면 "데이터 필요" 명시.
    """
)

# 김이사는 각 임원 응답을 기다린 후 종합
```

## Phase 3 — 시스템 위생 점검

브리핑 작성 **전**에 반드시 수행:

```markdown
| # | 점검 | Grep/Read 검증 | 이상 기준 |
|:---|:---|:---|:---|
| 1 | todo.md 정체 | 진행 중 항목 7일+ 무갱신 | 1건+ |
| 2 | lessons.md 기록 주기 | 마지막 교훈 날짜 | 14일+ 미기록 |
| 3 | predictions.md 검증 지연 | 검증일 지난 P-XXX | 1건+ |
| 4 | session_handoff.md 최신성 | 마지막 업데이트 | 7일+ |
| 5 | 파일 참조 깨짐 | 주요 경로 Grep | 1건+ |
```

이상 있으면 브리핑 하단에 "🔧 시스템 위생" 섹션 추가. 없으면 "시스템 위생: 정상 ✓" 한 줄.

  - 확신도: 70%
- **김이사 추천**: **옵션 B** — 이유: 2기 성공 사례가 책의 최고 마케팅 자산. 먼저 2기 성공 만들고 그걸로 책 완성.

## 이 파이프라인이 실패하는 3가지 시나리오 (Inversion)

1. **Tier 2 소집 남발 → 5분 브리핑이 30분 됨** → 조기 경고: 소집 결정 로직 준수 확인
2. **출처 없는 숫자 혼입 → Zero-Inference 위반** → 조기 경고: 출처 컬럼 강제, 빈 셀이면 "데이터 필요"
3. **위험 신호가 없다고 거짓 보고** → 조기 경고: 진짜 없으면 "위험 신호 없음. 🟢" 명시, 추측으로 만들지 말 것
